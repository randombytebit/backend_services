### Functional Requirements

- **Role**: System Designer
- **Requirement ID**: FR-01
- **Requirement Title**: Algorithm Selection for ML Solutions
- **Requirement Description**: The system must support the selection of appropriate machine learning algorithms based on problem type (e.g., classification, regression), algorithm characteristics (e.g., supervised, unsupervised), and specific algorithm types (e.g., regression, Bayesian) to meet desired outcomes.
- **Citation**: Section III, Page 3
- **Requirement Related Title**: NFRs for Machine Learning Example

- **Role**: System Designer
- **Requirement ID**: FR-02
- **Requirement Title**: Capture Algorithm Assumptions and Optimizations
- **Requirement Description**: The system must allow for capturing algorithm assumptions and optimizations (e.g., Naïve Bayesian independence assumption, Nearest Neighbor data projection) to evaluate their impact on solution performance.
- **Citation**: Section III, Page 3
- **Requirement Related Title**: NFRs for Machine Learning Example

- **Role**: System Designer
- **Requirement ID**: FR-03
- **Requirement Title**: Runtime Monitoring of ML Solutions
- **Requirement Description**: The system must enable runtime monitoring of machine learning solutions to assess the achievement of critical qualities during operation.
- **Citation**: Section IV, Challenge C4, Page 4
- **Requirement Related Title**: Challenges

- **Role**: System Designer
- **Requirement ID**: FR-04
- **Requirement Title**: Support for ML Solution Evolution
- **Requirement Description**: The system must support evolution of ML solutions by facilitating retraining or modification based on changes in available training data or quality requirements.
- **Citation**: Section IV, Challenge C6, Page 4
- **Requirement Related Title**: Challenges

- **Role**: System Designer
- **Requirement ID**: FR-05
- **Requirement Title**: Integration of ML with Traditional Software
- **Requirement Description**: The system must support the integration of ML components with traditional software components to form holistic solutions, considering quality perspectives across both.
- **Citation**: Section IV, Challenge C7, Page 4
- **Requirement Related Title**: Challenges

- **Role**: System Designer
- **Requirement ID**: FR-06
- **Requirement Title**: Language for Expressing NFRs in ML Context
- **Requirement Description**: The system must provide a language or representation to express and specify quality requirements for ML solutions, including targets, trade-offs, and domain context influences.
- **Citation**: Section IV, Challenge C5, Page 4
- **Requirement Related Title**: Challenges

- **Role**: System Designer
- **Requirement ID**: FR-07
- **Requirement Title**: Reasoning Over NFRs at Design Time
- **Requirement Description**: The system must support reasoning over non-functional requirements at design time to make context-based trade-offs and inform implementation decisions for ML solutions.

- **Role**: System Designer
- **Requirement ID**: FR-08
- **Requirement Title**: Reasoning Over NFRs at Runtime
- **Requirement Description**: The system must support runtime reasoning over non-functional requirements to monitor and assess the satisfaction of critical qualities as the ML solution operates.

- **Requirement ID**: FR-09
- **Requirement Title**: Handling Changing Data in ML Solutions

### Non-Functional Requirements

- **Role**: System Designer
- **Requirement ID**: NF-01
- **Requirement Title**: Fairness in ML Solutions
- **Requirement Description**: ML solutions must ensure fairness by mitigating unintended biases in decision-making processes, considering context-specific definitions and trade-offs with other qualities like performance or accuracy.
- **Citation**: Section I, Page 1; Section II.B, Page 2; Section IV, Challenge C1, Page 4
- **Requirement Related Title**: Introduction; Qualities for Machine Learning; Challenges

- **Role**: System Designer
- **Requirement ID**: NF-02
- **Requirement Title**: Transparency in ML Solutions
- **Requirement Description**: ML solutions must provide transparency by explaining how results are derived to build trust and mitigate issues in understanding decision processes.
- **Citation**: Section I, Page 1; Section II.B, Page 3
- **Requirement Related Title**: Introduction; Qualities for Machine Learning

- **Role**: System Designer
- **Requirement ID**: NF-03
- **Requirement Title**: Security in ML Solutions

- **Role**: System Designer
- **Requirement ID**: NF-04
- **Requirement Title**: Privacy in ML Solutions
