### Non-Functional Requirements

- **Role**: System Developer
- **Requirement ID**: NF-01
- **Requirement Title**: Usability Settings
- **Requirement Description**: The system must ensure usability by allowing users to create personal profiles, configure microphone and camera settings, select measurement units (e.g., centimeters, meters for height), make video calls with other participants, select conversation starters, and log out.
- **Citation**: Table 2, Page 5
- **Requirement Related Title**: Usability

- **Role**: System Developer
- **Requirement ID**: NF-02
- **Requirement Title**: Accessibility Features
- **Requirement Description**: The system must be accessible by allowing users to save and mark appointments, reminders, notes, recorded voices, photos, and messages; send feedback on consultant performance; select text from a default list; and share individual lessons, health record information with doctors, and patient-consultant data exchange.
- **Citation**: Table 2, Page 5
- **Requirement Related Title**: Accessibility

- **Role**: System Developer
- **Requirement ID**: NF-03
- **Requirement Title**: Compatibility with Insurance Data
- **Requirement Description**: The system must be compatible with recording insurance name and number in the required service format.
- **Citation**: Table 2, Page 5
- **Requirement Related Title**: Compatibility
