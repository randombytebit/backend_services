### Functional Requirements

**Functional Requirement 1**  
- **Role**: System  
- **Requirement ID**: FR-01  
- **Title**: Authorized Use of Sensors or Portals  
- **Description**: The system shall demand that applications explicitly request permission from the user to access sensors and portals (e.g., microphone, camera). The system shall also provide settings to manage these permissions, allowing users to grant or revoke access as needed.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table I.  
- **Related Title**: Smartphone Privacy Management System, IoT Device Access Control System, Wearable Tech Sensor Authorization System  

**Functional Requirement 2**  
- **Role**: System  
- **Requirement ID**: FR-02  
- **Title**: Avoidance of Privacy Leakage in User Behavior Information Collection  
- **Description**: The system shall notify users specifically and distinctly about the types of user information needed for behavior analysis. It shall strictly prohibit the collection of the most sensitive user information (e.g., passwords) without explicit permission.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table II.  
- **Related Title**: Mobile App Data Collection System, Behavioral Analytics Privacy System, Smart Device Usage Tracking System  

**Functional Requirement 3**  
- **Role**: System  
- **Requirement ID**: FR-03  
- **Title**: Guard for Personal Mobile Data  
- **Description**: The system shall provide warnings to the user when an application intends to access personal files, data, or credentials (e.g., messages, contacts). Access shall only be granted if the user explicitly permits it, and the system shall allow users to withdraw or regrant permissions at any time.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table III.  
- **Related Title**: Mobile Data Protection System, Personal Information Security System, Tablet Privacy Control System  

**Functional Requirement 4**  
- **Role**: System  
- **Requirement ID**: FR-04  
- **Title**: Privacy Protection over Mobile Cloud Services  
- **Description**: The system shall ensure confidentiality of data backup by performing encryption before sending data over the Internet. Secure tokens shall be used for authentication, and users shall be informed about what data is backed up to the cloud, with the ability to stop backups or delete data from the cloud at any time.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table IV.  
- **Related Title**: Cloud Storage Privacy System, Mobile Backup Security System, IoT Cloud Data Protection System  

**Functional Requirement 5**  
- **Role**: System  
- **Requirement ID**: FR-05  
- **Title**: Authentication of Mobile Users  
- **Description**: The system shall provide authentication mechanisms for the device, folders, applications, and privacy-sensitive data to prevent unauthorized access. This includes strong encryption algorithms for passwords and appropriate length requirements.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table V.  
- **Related Title**: Mobile Device Authentication System, Smartphone Lock Security System, Wearable Device Access Control System  

**Functional Requirement 6**  
- **Role**: System  
- **Requirement ID**: FR-06  
- **Title**: Financial Information Protection  
- **Description**: The system shall strictly prevent applications from permanently storing users’ financial information locally. Applications shall not transmit financial information to invalid or unauthenticated parties, and all data transmission must be secure.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table VI.  
- **Related Title**: Mobile Payment Security System, E-Banking Privacy System, Digital Wallet Protection System  

**Functional Requirement 7**  
- **Role**: System  
- **Requirement ID**: FR-07  
- **Title**: Mobile Communication Secrecy  
- **Description**: The system shall mandate that applications transmit privacy-sensitive data through secure channels (e.g., SSL protocol) to prevent interception during communication.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table VII.  
- **Related Title**: Secure Messaging System, Mobile Data Transmission Security System, IoT Communication Privacy System

### Non-Functional Requirements

**Non-Functional Requirement 1**  
- **Role**: System  
- **Requirement ID**: NFR-01  
- **Title**: Usability vs. Security Tradeoff in Authentication  
- **Description**: The system shall balance usability and security in authentication mechanisms, ensuring that security measures (e.g., password length, encryption strength) do not overly hinder user experience while still protecting against unauthorized access.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table V.  
- **Related Title**: Mobile Device Usability System, Smartphone Security Balance System, Wearable Tech Authentication System  

**Non-Functional Requirement 2**  
- **Role**: System  
- **Requirement ID**: NFR-02  
- **Title**: Confidentiality in Cloud Data Backup  
- **Description**: The system shall ensure high confidentiality for data backed up to cloud services through mandatory encryption and secure authentication tokens, protecting user privacy even during automated backups.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table IV.  
- **Related Title**: Cloud Backup Confidentiality System, Mobile Data Privacy System, IoT Cloud Security System  

**Non-Functional Requirement 3**  
- **Role**: System  
- **Requirement ID**: NFR-03  
- **Title**: Security in Mobile Communication  
- **Description**: The system shall enforce the use of secure transport techniques for communication between the device and servers, ensuring privacy-sensitive data is protected from interception, even at the cost of increased development complexity.  
- **Citation**: Xiao Xuan, Ye Wang, and Shanping Li. (2014). Privacy Requirements Patterns for Mobile Operating Systems. RePa 2014, Karlskrona, Sweden, Table VII.  
- **Related Title**: Secure Mobile Communication System, Encrypted Data Transfer System, IoT Network Security System
