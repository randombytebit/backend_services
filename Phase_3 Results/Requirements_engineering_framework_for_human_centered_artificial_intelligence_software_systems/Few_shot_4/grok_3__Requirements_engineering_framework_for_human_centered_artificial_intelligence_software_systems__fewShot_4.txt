**Functional Requirement**  
**Role:** System  
**Requirement ID:** FR-01  
**Title:** User Needs Identification  
**Description:** The system must identify and document user needs to ensure that AI-based software addresses specific human-centered requirements, including who the users are, why the system is needed, what it is used for, and how users interact with it (proactive or reactive, visible or invisible features). This includes evaluating whether the system should automate or augment user tasks and specifying the reward function for decision-making. [cite: Section 2.1, Page 2]  
**Citation:** Ahmad, K., Abdelrazek, M., Arora, C., Baniya, A. A., Bano, M., & Grundy, J. (2023). Requirements engineering framework for human-centered artificial intelligence software systems. Applied Soft Computing, 143, 110455. [cite: Section 2.1, Page 2]  
**Related Title:** Personalized Healthcare AI System, Adaptive Learning Platform for Education [cite: Section 2.1, Page 2]

**Functional Requirement**  
**Role:** System  
**Requirement ID:** FR-02  
**Title:** Model Needs Specification  
**Description:** The system must specify model needs by selecting appropriate machine learning types (supervised, unsupervised, reinforcement learning), optimizing for specific attributes (accuracy, explainability, robustness), and defining training approaches (static or dynamic). It must also include model tuning based on user feedback, parameter adjustments, and scalability considerations. [cite: Section 2.2, Page 3]  
**Citation:** Ahmad, K., Abdelrazek, M., Arora, C., Baniya, A. A., Bano, M., & Grundy, J. (2023). Requirements engineering framework for human-centered artificial intelligence software systems. Applied Soft Computing, 143, 110455. [cite: Section 2.2, Page 3]  
**Related Title:** Autonomous Vehicle Navigation System, Financial Fraud Detection AI [cite: Section 2.2, Page 3]

**Functional Requirement**  
**Role:** System  
**Requirement ID:** FR-03  
**Title:** Data Needs Collection and Selection  
**Description:** The system must define data needs by identifying the type, amount, and diversity of data required for training and testing AI models. It must specify data selection criteria (features, labels, examples, sampling rates) and collection methods while ensuring data sources are responsibly sourced and up-to-date. [cite: Section 2.3, Page 3]  
**Citation:** Ahmad, K., Abdelrazek, M., Arora, C., Baniya, A. A., Bano, M., & Grundy, J. (2023). Requirements engineering framework for human-centered artificial intelligence software systems. Applied Soft Computing, 143, 110455. [cite: Section 2.3, Page 3]  
**Related Title:** Genomic Data Analysis Platform, Real-Time Traffic Monitoring System [cite: Section 2.3, Page 3]

**Functional Requirement**  
**Role:** System  
**Requirement ID:** FR-04  
**Title:** Feedback and User Control Mechanisms  
**Description:** The system must establish mechanisms for collecting user feedback (implicit, explicit, calibration) and integrating it into model tuning. It must provide users with control options, allowing them to adjust preferences, dismiss feedback, or opt-out, ensuring alignment with user mental models and privacy considerations. [cite: Section 2.4, Page 3]  
**Citation:** Ahmad, K., Abdelrazek, M., Arora, C., Baniya, A. A., Bano, M., & Grundy, J. (2023). Requirements engineering framework for human-centered artificial intelligence software systems. Applied Soft Computing, 143, 110455. [cite: Section 2.4, Page 3]  
**Related Title:** Smart Home Automation System, Customer Support Chatbot Interface [cite: Section 2.4, Page 3]

**Non-Functional Requirement**  
**Role:** System  
**Requirement ID:** NFR-01  
**Title:** Explainability and Trust  
**Description:** The system must ensure explainability by providing clear explanations of AI predictions, limitations, and capabilities to both users and stakeholders. It must inform users about data usage, system changes, and consequences of actions, while displaying confidence levels appropriately to avoid over-trust and comply with legal requirements such as GDPR. [cite: Section 2.5, Page 3-4]  
**Citation:** Ahmad, K., Abdelrazek, M., Arora, C., Baniya, A. A., Bano, M., & Grundy, J. (2023). Requirements engineering framework for human-centered artificial intelligence software systems. Applied Soft Computing, 143, 110455. [cite: Section 2.5, Page 3-4]  
**Related Title:** Clinical Decision Support System, Legal Compliance AI Auditor [cite: Section 2.5, Page 3-4]

**Non-Functional Requirement**  
**Role:** System  
**Requirement ID:** NFR-02  
**Title:** Error and Failure Management  
**Description:** The system must identify and mitigate errors and failures in AI-based software by specifying error types (context, background, system limitations) and sources (prediction, data, input, output). It must provide action plans to address errors, including user-perceived errors and system integration issues, to ensure reliability in high-stakes scenarios. [cite: Section 2.6, Page 4]  
**Citation:** Ahmad, K., Abdelrazek, M., Arora, C., Baniya, A. A., Bano, M., & Grundy, J. (2023). Requirements engineering framework for human-centered artificial intelligence software systems. Applied Soft Computing, 143, 110455. [cite: Section 2.6, Page 4]  
**Related Title:** Aerospace Control System AI, Emergency Response AI Network [cite: Section 2.6, Page 4]

**Non-Functional Requirement**  
**Role:** System  
**Requirement ID:** NFR-03  
**Title:** Data Quality and Fairness  
**Description:** The system must ensure data quality by maintaining accuracy, completeness, consistency, credibility, and currentness of data used in AI models. It must address fairness by identifying and mitigating biases (automation, selection, group attribution) to prevent discriminatory outcomes and ensure diverse representation in datasets. [cite: Section 2.3, Page 3]  
**Citation:** Ahmad, K., Abdelrazek, M., Arora, C., Baniya, A. A., Bano, M., & Grundy, J. (2023). Requirements engineering framework for human-centered artificial intelligence software systems. Applied Soft Computing, 143, 110455. [cite: Section 2.3, Page 3]  
**Related Title:** Bias-Free Recruitment AI Tool, Equitable Public Policy AI System [cite: Section 2.3, Page 3]

**Non-Functional Requirement**  
**Role:** System  
**Requirement ID:** NFR-04  
**Title:** Privacy and Security Compliance  
**Description:** The system must protect personal information and comply with privacy laws and regulations when handling user data and feedback. It must ensure that data usage and sharing practices are transparent and secure, safeguarding user privacy during interactions with the AI system. [cite: Section 2.3 & 2.4, Page 3]
