Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Authorized Use of Sensors or Portals  
Description: The system shall demand the application to explicitly ask for permission from the user to access the sensors and portals. The system shall also provide a host of settings to manage these permissions.  
Citation: Xuan, X., Wang, Y., & Li, S. (2014). Privacy Requirements Patterns for Mobile Operating Systems. Proceedings of the 2nd International Workshop on Requirements Patterns (RePa), Karlskrona, Sweden.  
Related Title: Biometric Authentication in Secure Access Control Systems, Location Tracking in Emergency Response Applications  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-02  
Title: Avoidance of Privacy Leakage in User Behavior Information Collection  
Description: The system shall notice user specifically and distinctly what kinds of user information is needed for behavior analysis. The most sensitive user information (e.g. password) shall be strictly prohibited to touch and collect.  
Citation: Xuan, X., Wang, Y., & Li, S. (2014). Privacy Requirements Patterns for Mobile Operating Systems. Proceedings of the 2nd International Workshop on Requirements Patterns (RePa), Karlskrona, Sweden.  
Related Title: Behavioral Analytics in Healthcare Monitoring Apps, User Profiling in Financial Fraud Detection Systems  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-03  
Title: Guard for Personal Mobile Data  
Description: The system shall provide warnings to the user when an application intends to access user files, data and credentials. If and only if the user has granted the permission, the application can touch the corresponding resources. The user can withdraw or regrant any permissions in somewhere of the system later.  
Citation: Xuan, X., Wang, Y., & Li, S. (2014). Privacy Requirements Patterns for Mobile Operating Systems. Proceedings of the 2nd International Workshop on Requirements Patterns (RePa), Karlskrona, Sweden.  
Related Title: Contact Management in Enterprise CRM Systems, Photo Privacy in Social Media Platforms  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-04  
Title: Privacy Protection over Mobile Cloud Services  
Description: The system shall ensure confidentiality of data backup. The data encryption must be performed before the data sending over the Internet. Secure tokens shall be used for authentication. The user should be aware what data will be backed up to the cloud. The user also can stop the backup function, and delete unwanted data from cloud with her device any time.  
Citation: Xuan, X., Wang, Y., & Li, S. (2014). Privacy Requirements Patterns for Mobile Operating Systems. Proceedings of the 2nd International Workshop on Requirements Patterns (RePa), Karlskrona, Sweden.  
Related Title: Data Synchronization in Remote Medical Record Systems, Backup Security in Corporate Cloud Storage  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-05  
Title: Authentication of Mobile Users  
Description: The system shall provide authentication for the device, folders, applications and privacy-sensitive data. The encryption algorithm of password shall be strong. The length requirement shall be set appropriately.  
Citation: Xuan, X., Wang, Y., & Li, S. (2014). Privacy Requirements Patterns for Mobile Operating Systems. Proceedings of the 2nd International Workshop on Requirements Patterns (RePa), Karlskrona, Sweden.  
Related Title: Device Lock in Military Communication Devices, Biometric Access in Government Secure Networks  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-06  
Title: Financial Information Protection  
Description: The system shall strictly prevent any application to store the user’s financial information permanently in local. The application cannot transmit user financial information to any invalid or unauthenticated parties. The credentials and data must be transmitted in a secure way.  
Citation: Xuan, X., Wang, Y., & Li, S. (2014). Privacy Requirements Patterns for Mobile Operating Systems. Proceedings of the 2nd International Workshop on Requirements Patterns (RePa), Karlskrona, Sweden.  
Related Title: Mobile Banking in High-Value Transaction Platforms, Payment Security in E-Commerce Defense Systems

Role: System
