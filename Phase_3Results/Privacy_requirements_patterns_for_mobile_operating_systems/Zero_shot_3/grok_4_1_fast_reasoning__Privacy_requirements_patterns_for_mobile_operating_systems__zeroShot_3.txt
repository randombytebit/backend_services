REQUIREMENT REPORT
- Role: Mobile Operating System
- Requirement ID: FR-01
- Requirement Title: Authorized use of sensors or portals
- Requirement Description: The system shall demand the application to explicitly ask for permission from the user to access the sensors and portals. The system shall also provide a host of settings to manage these permissions.
- Citation: Table I, Page 3
- Requirement Related Title: Privacy Pattern 1
- Role: Mobile Operating System
- Requirement ID: FR-02
- Requirement Title: Avoidance of Privacy Leakage in User Behavior Information Collection
- Requirement Description: The system shall notice user specifically and distinctly what kinds of user information is needed for behavior analysis. The most sensitive user information (e.g. password) shall be strictly prohibited to touch and collect.
- Citation: Table II, Page 3
- Requirement Related Title: Privacy Pattern 2
- Role: Mobile Operating System
- Requirement ID: FR-03
- Requirement Title: Guard for Personal Mobile Data
- Requirement Description: The system shall provide warnings to the user when an application intend to access user files, data and credentials. If and only if the user has granted the permission, the application can touch the corresponding resources. The user can withdraw or regrant any permissions in somewhere of the system later.
- Citation: Table III, Page 3
- Requirement Related Title: Privacy Pattern 3
- Role: Mobile Operating System
- Requirement ID: FR-04
- Requirement Title: Privacy Protection over Mobile Cloud Services
- Requirement Description: The system shall ensure confidentiality of data backup. The data encryption must be performed before the data sending over the Internet. Secure tokens shall be used for authentication. The user should be awared what data will be backed up to the cloud. The user also can stop the backup function, and delete unwanted data from cloud with her device any time.
- Citation: Table IV, Page 4
- Requirement Related Title: Privacy Pattern 4
- Role: Mobile Operating System
- Requirement ID: FR-05
- Requirement Title: Authentication of Mobile Users
- Requirement Description: The system shall provide authentication for the device, folders, applications and privacy-sensitive data. The encryption algorithm of password shall be strong. The longth requirment shall be set appropriately.
- Citation: Table V, Page 4
- Requirement Related Title: Privacy Pattern 5
- Role: Mobile Operating System
- Requirement ID: FR-06
- Requirement Title: Financial Information Protection
- Requirement Description: The system shall strictly prevent any application to store the user’s financial information permanently in local. The application cannot transmit user financial information to any invalid or unauthenticated parties. The credentials and data must be transmited in a secure way.
- Citation: Table VI, Page 4
- Requirement Related Title: Privacy Pattern 6
- Role: Mobile Operating System
- Requirement ID: FR-07
- Requirement Title: Mobile Communication Secrecy
- Requirement Description: The system shall request the application transmit privacy-sensitive data with secure channel forcibly.
- Citation: Table VII, Page 4
- Requirement Related Title: Privacy Pattern 7
