Non-Functional Requirement
Role: System
Requirement ID: NFR-01
Title: Security Against Authentication Threats
Description: The system must defend against common authentication threats such as password leaks, phishing attacks, brute-force attacks, replay attacks, and man-in-the-middle attacks by implementing passwordless mechanisms like Passkey with public-private key cryptography, challenge-response protocols, and fallback to TOTP with rate limiting, while ensuring robust protection through technologies like bcrypt for hashing, JWT blacklisting, and WebAuthn standards.
Citation: Tran, L. et al. (2025). The Passwordless Authentication with Passkey Technology from an Implementation Perspective. [Pages 1-6, e.g., Abstract, Section II Literature Review, Section III Methodology, Section IV Experimental Setup, Section V Discussion]
Related Title: Banking Authentication Systems, Healthcare Data Access Controls, Government Identity Verification Platforms

Non-Functional Requirement
Role: System
Requirement ID: NFR-02
Title: Usability and User Experience
Description: The system must provide a seamless and minimal-complexity user experience for authentication, including multi-device synchronization, biometric integration, and smooth transitions from password-based or TOTP methods to Passkey, while addressing user hesitations such as account recovery limitations and ensuring clear onboarding processes.

Non-Functional Requirement
Role: System
Requirement ID: NFR-03
Title: Performance and Scalability
Description: The system must maintain high performance and scalability through efficient caching with Redis, secure session management, rate limiting to prevent brute-force attacks, and modular architecture using Docker for consistent deployment, while minimizing computational load and ensuring quick response times for authentication processes.

Functional Requirement
Role: System
Requirement ID: FR-01
Title: Passwordless User Authentication via Passkey
Description: The system must enable users to register and authenticate using Passkey technology, involving the generation of WebAuthn registration and authentication options, storage of credentials (ID, public key, counter, device name), verification of biometric or security key inputs, and issuance of JWT tokens upon successful authentication, integrated with front-end and back-end components for a passwordless experience.
Citation: Tran, L. et al. (2025). The Passwordless Authentication with Passkey Technology from an Implementation Perspective. [Pages 1-6, e.g., Section III Methodology, Section IV Experimental Setup, Section IV Logic Implementation]
Related Title: Biometric Access Control in Secure Facilities, Passwordless Login for Financial Apps, Device-Based Authentication in IoT Networks

Functional Requirement
Role: System
Requirement ID: FR-02
Title: Hybrid Authentication with TOTP Backup
Description: The system must support TOTP as a secondary authentication mechanism during registration and as a backup for login, generating unique secret keys via speakeasy library, verifying time-based codes, and integrating with Passkey for a hybrid approach that prioritizes Passkey while allowing fallback to mitigate timing synchronization issues and ensure compatibility.
Citation: Tran, L. et al. (2025). The Passwordless Authentication with Passkey Technology from an Implementation Perspective. [Pages 1-6, e.g., Section III Methodology, Section IV Experimental Setup, Section IV Logic Implementation, Section V Discussion]
Related Title: Two-Factor Authentication in Email Services, Backup Login for Critical Infrastructure Systems, Transitional Authentication in Legacy Enterprise Networks

Functional Requirement
Role: System
Requirement ID: FR-03
Title: Secure User Registration and Verification
Description: The system must facilitate user registration by generating and sending email-based verification codes via Nodemailer, storing temporary data in Redis with expiration, hashing passwords with bcrypt, and verifying codes to create accounts in MongoDB, while supporting OAuth2 integration for third-party logins like Google to retrieve and store user profiles.
Citation: Tran, L. et al. (2025). The Passwordless Authentication with Passkey Technology from an Implementation Perspective. [Pages 1-6, e.g., Section III Methodology, Section IV Experimental Setup, Section IV Logic Implementation]
Related Title: Account Creation in Social Media Platforms, Verification Processes in Online Banking, Registration Flows in E-Learning Systems
