Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Brute Force Protection  
Description: Passkey uses a unique challenge-response mechanism without passwords to prevent brute-force attacks, supplemented by rate limiting with express-rate-limit.  
Citation: Tran et al. (2025). The Passwordless Authentication with Passkey Technology from an Implementation Perspective. Page 3 (Sec III.A), Page 5 (Sec IV.D).  
Related Title: Financial Transaction Systems, High-Security Web Applications

Role: System  
Requirement ID: NFR-02

Citation: Tran et al. (2025). The Passwordless Authentication with Passkey Technology from an Implementation Perspective. Page 3 (Sec III.A).

Role: System  
Requirement ID: NFR-03  
Title: Replay Attack Prevention  
Description: Passkey generates a new random challenge per authentication attempt combined with timestamped signatures, and updates credential counter to neutralize replay attacks.  
Citation: Tran et al. (2025). The Passwordless Authentication with Passkey Technology from an Implementation Perspective. Page 3 (Sec III.A), Page 5 (Sec IV.C.5).  
Related Title: Secure API Gateways, IoT Device Authentication

Role: System  
Requirement ID: NFR-04  
Title: Password Hashing Security  
Description: Use Bcrypt with salt and configurable work factor (e.g., 10 salt rounds) for irreversible hashing of passwords to resist brute-force and dictionary attacks.  
Citation: Tran et al. (2025). The Passwordless Authentication with Passkey Technology from an Implementation Perspective. Page 3 (Sec III.D), Page 4 (Sec III.E).  
Related Title: User Management Databases, Legacy Password Migration Systems
