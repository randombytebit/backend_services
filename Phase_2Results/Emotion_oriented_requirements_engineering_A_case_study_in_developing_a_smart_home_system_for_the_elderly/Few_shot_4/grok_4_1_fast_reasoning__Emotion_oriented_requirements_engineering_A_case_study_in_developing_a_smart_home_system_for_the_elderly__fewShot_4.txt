Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Privacy Preservation  
Description: System must use non-intrusive motion sensors to monitor activities without violating user privacy (e.g., no cameras or wearables that evoke feeling old), store sensor data locally within the hub, and secure data to prevent hacking or unauthorized access.  
Citation: Page 2 (non-intrusive sensors for privacy), Page 7 (privacy secured emotional goal, data kept local). Curumsing et al. (2019). Emotion-oriented requirements engineering: A case study in developing a smart home system for the elderly.  
Related Title: Telemedicine Patient Monitoring Systems, Military Personnel Health Trackers  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-02  
Title: User Safety Assurance  
Description: System must enable users to feel safe by accurately detecting anomalies (e.g., falls, irregular bathroom visits) and escalating them to carers or services, based on a defined monitoring model.  
Citation: Page 7 (Safe emotional goal linked to detect anomaly). Curumsing et al. (2019).  
Related Title: Hospital Fall Detection Systems, Critical Care Unit Alarms  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-03  
Title: System Usability and Integration  
Description: System must integrate seamlessly into users' daily routines to feel non-obtrusive, comfortable, and not overwhelming (e.g., limited message types like morning greetings and reminders to avoid frustration).  
Citation: Page 7 (Integrated, Comfortable emotional goals), Page 10 (integrated into life, waited for messages). Curumsing et al. (2019).  
Related Title: Ambient Assisted Living Platforms, Nuclear Plant Operator Interfaces  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-04  
Title: User Independence and Control  
Description: System must support user independence and sense of control by enabling self-living without feeling monitored, dependent, or controlled (e.g., passive monitoring, user-configurable reminders).  
Citation: Page 2 (emotional needs: independent, secure), Page 7 (Independent, In control emotional goals). Curumsing et al. (2019).  
Related Title: Aviation Cockpit Assistance Systems, Autonomous Vehicle User Overrides  

Functional Requirement  
Role: System  
Requirement ID: FR-01  
Title: Passive Environment Monitoring  
Description: System must continuously process real-world sensor inputs (motion, door contact, temperature, humidity) for passive monitoring of elderly activities in the home.  
Citation: Page 8 (sensor data collection), Page 7 (SofiHub features). Curumsing et al. (2019).  
Related Title: Battlefield Soldier Tracking Systems, Oil Rig Safety Monitoring  

Functional Requirement  
Role: System  
Requirement ID: FR-02  
Title: Routine Learning and Anomaly Detection  
Description: System must learn user routines and behaviors via AI from sensor data and detect anomalies (e.g., abnormal bed time, open doors, falls) to trigger alerts.  
Citation: Page 7 (learn user’s routine and behaviour, detect anomaly). Curumsing et al. (2019).  
Related Title: Air Traffic Control Anomaly Detectors, Submarine Crew Health Monitors  

Functional Requirement  
Role: System  
Requirement ID: FR-03  
Title: Notification and Reminder Issuance  
Description: System must issue user-specific reminders and notifications (e.g., hydration, medication, appointments) triggered by learned routines or activity.  
Citation: Page 7 (issue notifications), Page 8 (reminders). Curumsing et al. (2019).  
Related Title: Mission-Critical Pilot Alert Systems, Surgical Robot Procedure Reminders  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-05  
Title: Carer Reassurance  
Description: System must reassure carers/relatives about elderly wellbeing through activity reports, customizable monitoring UI, and anomaly escalations.  
Citation: Page 7 (Reassured emotional goal for carers/relatives). Curumsing et al. (2019).  
Related Title: Remote Drone Operator Dashboards, Space Mission Control Consoles
