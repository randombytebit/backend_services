### Architectural Reasoning:
- **Component Identification**: The source text describes SofiHub as a smart home system with core functional capabilities including passive monitoring via non-intrusive motion/contact sensors, AI-driven learning of user routines/behavior, issuing activity-triggered reminders (e.g., hydration, medication, appointments), anomaly detection (e.g., abnormal bed time, falls, open doors), alerts/escalation, environmental automation (heating/cooling), and a carer UI for activity monitoring/reports. These are explicitly detailed in Sections 4.1 (SofiHub features), 4.2 (goal model: "support independent living" decomposed to "learn user’s routine and behaviour", "issue notifications", "detect anomaly"), and trials (Page 7-10).
- **Constraint Identification**: Quality constraints include emotional goals (e.g., feel safe, independent, in control, integrated, comfortable, empowered, privacy secured, reassured for carers/relatives, less lonely, cared about) modeled as NFRs linked to FRs (e.g., safe/reassured via anomaly detection; independent/in control via limited notifications; privacy via local data storage). Traditional NFRs: non-intrusive (no wearables), secure data (local storage, authorized access only), reliable (power concerns noted), integrated into routine (to avoid frustration). These are verified in emotion/goal models (Figs 4-5,7; Page 5-7,10), trials (Pages 8-12: questionnaires/interviews confirm emotional impacts), and refinements (new goals like "less lonely").
- **Functional Logic (FRs)**: Role is 'System' performing sensing/monitoring/reminders/alerts/AI learning; consistent with elderly smart home design for independent living (e.g., Pages 1-2,4).
- **Quality Logic (NFRs)**: 'System' must satisfy emotional/perceptual constraints (e.g., users perceive system as non-intrusive/privacy-secure) and report via metrics (questionnaires, interviews); linked to FRs (e.g., anomaly detection -> feel safe).
- **Domain Mapping**: FRs apply to elderly care (e.g., fall detection, routine monitoring); NFRs to socio-technical systems (e.g., privacy in IoT homes, emotional adoption in aged care). Extraction limited to explicit, verifiable requirements; no invention. 5 FRs/5 NFRs selected as core, non-redundant from goal model/trials.
