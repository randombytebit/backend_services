Functional Requirement  
Role: System  
Requirement ID: FR-01  
Title: Learn User Routine and Behaviour  
Description: System should learn user’s routine and behaviour (e.g., via AI capability) as a sub-goal of supporting independent living, with further decomposition into issuing notifications based on learned patterns.  
Citation: Page 6 (goal model description); Page 7 (SofiHub features: "built-in AI capability that can learn and adapt to a users behaviour").  
Related Title: Elderly Smart Home Behavior Learning, AI Adaptation in Ambient Assisted Living  

Functional Requirement  
Role: System  
Requirement ID: FR-02  
Title: Detect Anomaly  
Description: System should detect anomalies (e.g., user spending an abnormal time in bed or a fall) to contribute to emotional goals like feeling safe, with escalation to service provider or carer.  
Citation: Page 5 (goal model: emotional goal "safe" linked to "detect anomaly"); Page 7 (SofiHub features: "issuing alerts when anomalous behaviour is detected (eg: user is spending an abnormal time in bed)").  
Related Title: Anomaly Detection for Elderly Falls, Daily Routine Deviation Monitoring  

Functional Requirement  
Role: System  
Requirement ID: FR-03  
Title: Issue Notifications and Alerts  
Description: System should issue user-specific notifications and alerts (e.g., hydration, medication reminders triggered by activity; anomaly alerts) based on learned routines and detected anomalies.  
Citation: Page 7 (SofiHub features: "issuing user specific reminders such as appointments... hydration and medication reminders... issuing alerts when anomalous behaviour is detected").  
Related Title: Activity-Triggered Reminders in Smart Homes, Elderly Anomaly Alert Systems  

Functional Requirement  
Role: System  
Requirement ID: FR-04  
Title: Sensor Data Collection Pipeline  
Description: System should collect and locally store sensor data (motion, door contact, air temperature, humidity) from non-intrusive sensors for AI learning, anomaly detection, and periodic retrieval as data dumps for analysis.  
Citation: Page 8 (data collection: "Sensor data was collected from the motion and contact sensors... recorded motion, opening and closing... air temperature and humidity. The data was locally stored in the SofiHub... retrieve data dumps at selected periods").  
Related Title: IoT Sensor Pipelines for Home Monitoring, Elderly Passive Sensing Dataflows  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: AI Component Accuracy Testing  
Description: System should support testing the accuracy of sensors and AI component predictions (e.g., routine learning, anomaly detection) using collected sensor data cross-checked against daily checklists for ground-truth validation.  
Citation: Page 8 (data collection: "The main uses of sensor data were to test the accuracy of sensors and the AI component of SofiHub... cross checked against the data recorded by the sensors during the weekly... to establish ground-truth to verify the results of SofiHub’s AI component").  
Related Title: ML Accuracy Validation in Wearable-Free Monitoring, Reliability Testing for Individual User Predictions
