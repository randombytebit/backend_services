Requirement ID: FR-01  
Requirement Title: Anomaly Detection and Alerting  
Requirement Description: System should passively monitor user activity using sensors to detect anomalous behaviors (such as falls, abnormal time spent in bed, or irregular routines) and automatically issue alerts or notifications to carers, relatives, or service providers to ensure timely emergency assistance.  
Requirement Related Title: Fall Detection Wearables, Elderly Emergency Response Systems, Smart Security Monitoring Apps, Home Intrusion Alert Software, Behavioral Anomaly Detectors in Assisted Living  

Requirement ID: FR-02  
Requirement Title: Reminder and Notification Functions  
Requirement Description: System should issue user-specific reminders for hydration, medication, appointments, special events, and daily routines, triggered by user activity or learned behavior patterns, while also providing notifications for security risks (e.g., doors left open) and automating environmental controls like heating or cooling.  
Requirement Related Title: Medication Adherence Apps, Smart Calendar Assistants, Hydration Tracking Devices, Appointment Reminder Software, Environmental Automation in IoT Homes  

Requirement ID: FR-03  
Requirement Title: User Behavior Learning and Monitoring  
Requirement Description: System should learn and adapt to the user's daily routines and behaviors through passive sensing (e.g., motion and contact sensors) to support independent living, consolidate activity data for reports, and enable carer access via a UI for monitoring and customizable summaries.  
Requirement Related Title: Activity Tracking Wearables, AI-Powered Habit Learning Systems, Remote Caregiver Dashboards, Routine Analysis in Smart Health Platforms, Personalized Assisted Living Software  

Requirement ID: NFR-01  
Requirement Title: Privacy and Data Security  
Requirement Description: System should ensure strict data isolation and security by keeping sensor data local to the hub, using non-intrusive ambient sensors to avoid privacy violations, and preventing unauthorized access to sensitive information, thereby making users feel their privacy is secured.  
Requirement Related Title: Secure IoT Ecosystems for Homes, Privacy-Centric Health Monitoring Platforms, Data-Protected Elderly Care Systems, Compliant Telehealth Solutions  

Requirement ID: NFR-02  
Requirement Title: Usability and Integration  
Requirement Description: System should be designed for seamless integration into daily life, with simple interactions (e.g., minimal messages to avoid overwhelming users), ease of use for non-technical elderly users, and features that promote feelings of control, comfort, and independence without requiring wearables or complex setups.  
Requirement Related Title: User-Friendly Smart Home Interfaces, Adaptive Elderly Tech Devices, Intuitive Ambient Assisted Living Systems, Non-Intrusive IoT Usability Platforms  

Requirement ID: NFR-03  
Requirement Title: Emotional Goal Support and Reliability  
Requirement Description: System should incorporate emotional goals such as reducing loneliness (e.g., through chatty messages), ensuring users feel safe, cared about, and reassured, while providing reliable operation (e.g., handling power failures gracefully) to avoid frustration and maintain trust in the system's performance.  
Requirement Related Title: Emotion-Aware AI Companions, Loneliness Mitigation Apps, Reliable Safety Nets in Aged Care Tech, Affective Computing in Smart Environments
