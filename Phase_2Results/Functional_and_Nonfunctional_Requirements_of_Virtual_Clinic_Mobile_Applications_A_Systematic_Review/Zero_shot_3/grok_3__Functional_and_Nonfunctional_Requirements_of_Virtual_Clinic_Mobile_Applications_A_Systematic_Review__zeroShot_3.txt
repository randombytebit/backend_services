**Non-Functional Requirements**

- **Role**: System Architect
- **Requirement ID**: NF-01
- **Requirement Title**: Usability - User Profile and Settings
- **Requirement Description**: The system must ensure usability by allowing users to create personal profiles, configure settings for microphone and camera, select measurement units (e.g., centimeters, meters for height), enable video calls with other participants, select conversation starters, and provide a logout option.
- **Citation**: Table 2, Page 5
- **Requirement Related Title**: Usability

- **Role**: System Architect
- **Requirement ID**: NF-02
- **Requirement Title**: Accessibility - Service Features
- **Requirement Description**: The system must be accessible by allowing users to save and mark appointments, reminders, notes, recorded voices, photos, and messages; send feedback on consultant performance; select text from a default list; and share individual lessons, health record information with doctors, and patient-consultant data exchange.
- **Citation**: Table 2, Page 5
- **Requirement Related Title**: Accessibility

- **Role**: System Architect
- **Requirement ID**: NF-03
- **Requirement Title**: Compatibility - Service Format
- **Requirement Description**: The system must be compatible with various formats, including the ability to record insurance name and number.
- **Citation**: Table 2, Page 5
- **Requirement Related Title**: Compatibility

- **Role**: System Architect
- **Requirement ID**: NF-04
- **Requirement Title**: Efficiency - Financial Transactions
- **Requirement Description**: The system must ensure efficiency by supporting financial transaction reporting.
- **Citation**: Table 2, Page 5
- **Requirement Related Title**: Efficiency

- **Role**: System Architect
- **Requirement ID**: NF-05
- **Requirement Title**: Security - Data Privacy and Integrity
- **Requirement Description**: The system must adhere to high security standards to protect data privacy and integrity during user interactions and data exchanges.
- **Citation**: Table 2, Page 5
- **Requirement Related Title**: Security

- **Role**: System Architect
- **Requirement ID**: NF-06
- **Requirement Title**: System Resilience and Robustness
- **Requirement Description**: The system must be designed to be resilient against potential failures and robust against adversarial threats to ensure continuous operation and protection of user data.
- **Citation**: Page 6, Discussion on UI and Security

- **Role**: System Architect
- **Requirement ID**: NF-07
- **Requirement Title**: Privacy Concerns Mitigation
- **Requirement Description**: The system must address privacy concerns by implementing measures to mitigate risks associated with inadequate smartphone skills, limited internet access, and fears of encountering fake healthcare professionals or patients.
- **Citation**: Page 5, Discussion on Privacy Concerns
- **Requirement Related Title**: Security

- **Requirement ID**: NF-08
- **Requirement Title**: Multilingual UI for International Expansion
- **Requirement Description**: The system must support a multilingual user interface to facilitate international expansion of specific VCMA services, ensuring accessibility across diverse user bases.
