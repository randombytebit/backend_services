Functional Requirement  
Role: System  
Requirement ID: FR-01  
Title: Demographic Data Documentation  
Description: System should document user demographic data including identity (first name and surname) and insurance details (insurance name and number).  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. [Page 4, Table 1]

Functional Requirement  
Role: System  
Requirement ID: FR-02  
Title: Health Record  
Description: System should manage health records including vital sign documentation (blood pressure, heart rate), kidney documentation (blood sugar), fitness documentation (weight, exercise, sleep, nutrition), skin documentation (condition of skin), history documentation (disease history, family history, allergy, medications history, vaccination, surgeries, smoking), present history documentation (symptoms and conditions, X-ray, tests, treatments, prescriptions, patient training), documentation of inpatient treatment history (date of admission and discharge, hospital name, chief complaint, summary), and medication documentation (name, dose, type, instruction, side effects).  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. [Page 4, Table 1]  
Related Title: Remote Patient Monitoring in Chronic Disease Management Apps [Page 4]

Functional Requirement  
Role: System  
Requirement ID: FR-03  
Title: General Features of the User Interface (UI)  
Description: System should provide general UI features including reminders (appointment, preventive care), display (patient’s self-declaration about health status, consultant identity with name, surname, and profile picture, curriculum vitae with expertise and consulting field, type of communication such as phone call, text, video call, and real-time, appointment details like consultation date, time, duration, and waiting time for response), service selection (choosing consultation day, time, and type of insurance), reporting (providing weekly/monthly reports on the patient’s treatment process, providing weekly/monthly reports of vital signs and medical history), support (documentation of communication ways like tagging capability and frequently asked questions, Q and A ability such as discussion forum with others including patients and consultants), and help/tutorial (text guide for help, training inside VCMA for working with it, about us, terms and conditions of use).  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. [Page 4, Table 1]  
Related Title: User Interaction in Virtual Consultation Systems [Page 4]

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Usability  
Description: System should ensure usability through settings including creating a personal profile for the user (consultant-patient), microphone, camera, measurement units selection (e.g., selection of measurement units like centimeters and meters for height), video call with other participants (making a video call with other connected individuals), selecting conversation starter, and log out.  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. [Page 5, Table 2]  
Related Title: Interface Design in Mobile Health Applications [Page 5]

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-02  
Title: Accessibility  
Description: System should ensure accessibility through services including save and mark (appointments, reminders, notes, recorded voices, photos, and messages; send feedback on consultant’s performance and select text from a default list) and ability to share (individual lessons, health record information with doctors, and patient-consultant data exchange).

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-05  
Title: Security  
Description: System should ensure security through login method (text password, email, activation code, and anonymously), verify the user’s identity by sending a verification code to the mobile phone, possibility of deleting consultation records, documents, and bank transactions, and encrypted patient data exchange.
