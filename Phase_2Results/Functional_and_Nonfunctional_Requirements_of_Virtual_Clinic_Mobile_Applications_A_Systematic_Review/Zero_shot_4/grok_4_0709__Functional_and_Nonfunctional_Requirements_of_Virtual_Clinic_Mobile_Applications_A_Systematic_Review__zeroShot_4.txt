REQUIREMENT REPORT

- Role: VCMA System
- Requirement ID: FR-01
- Requirement Title: Demographic Data Documentation
- Requirement Description: The system must support documentation of demographic data including identity (first name and surname) and insurance (insurance name and number).
- Citation: Zahra Parsaei et al., 2024, Table 1
- Requirement Related Title: Functional Requirements

- Role: VCMA System
- Requirement ID: FR-02
- Requirement Title: Health Record
- Requirement Description: The system must support health record management including vital sign documentation (blood pressure, heart rate), kidney documentation (blood sugar), fitness documentation (weight, exercise, sleep, nutrition), skin documentation (condition of skin), history documentation (disease history, family history, allergy, medications history, vaccination, surgeries, smoking), present history documentation (symptoms and conditions, X-ray, tests, treatments, prescriptions, patient training), documentation of inpatient treatment history (date of admission and discharge, hospital name, chief complaint, summary), medication documentation (name, dose, type, instruction, side effects).
- Citation: Zahra Parsaei et al., 2024, Table 1
- Requirement Related Title: Functional Requirements

- Role: VCMA System
- Requirement ID: FR-03
- Requirement Title: General Features of the User Interface (UI)
- Requirement Description: The system must provide general UI features including reminders (appointment, preventive care), display (patient’s self-declaration about his health status, consultant identity: name and surname and profile picture, curriculum vitae: expertise and consulting field, type of communication: phone call, text, video call, and real-time, appointment: consultation date, time, duration, and waiting time for response), service selection (choosing consultation day, time, and type of insurance), reporting (providing weekly/monthly reports on the patient’s treatment process, providing weekly/monthly reports of vital signs and medical history), support (documentation of communication ways: tagging capability in VCMA and frequently asked questions, Q and A ability: discussion forum with others (patients and consultants)), help/tutorial (text guide: help, training inside VCMA: working with it, about us, terms and conditions of use).
- Citation: Zahra Parsaei et al., 2024, Table 1
- Requirement Related Title: Functional Requirements

- Role: VCMA System
- Requirement ID: NF-01
- Requirement Title: Usability
- Requirement Description: The system must provide usability features including setting (creating a personal profile for the user (consultant-patient), microphone, camera, measurement units selection: selection of measurement units like centimeters and meters for height, video call with other participants: making a video call with other connected individuals, selecting conversation starter, log out).
- Citation: Zahra Parsaei et al., 2024, Table 2
- Requirement Related Title: Nonfunctional Requirements

- Role: VCMA System
- Requirement ID: NF-02
- Requirement Title: Accessibility
- Requirement Description: The system must provide accessibility features including service (save and mark: appointments, reminders, notes, recorded voices, photos, and messages; send feedback on consultant’s performance and select text from a default list, ability to share: individual lessons, health record information with doctors, and patient-consultant data exchange).
- Citation: Zahra Parsaei et al., 2024, Table 2
- Requirement Related Title: Nonfunctional Requirements

- Role: VCMA System
- Requirement ID: NF-03
- Requirement Title: Compatibility
- Requirement Description: The system must provide compatibility features including service format (insurance: recording name and number).
- Citation: Zahra Parsaei et al., 2024, Table 2
- Requirement Related Title: Nonfunctional Requirements

- Role: VCMA System
- Requirement ID: NF-04
- Requirement Title: Efficiency
- Requirement Description: The system must provide efficiency features including service (financial transaction reporting).
- Citation: Zahra Parsaei et al., 2024, Table 2
- Requirement Related Title: Nonfunctional Requirements

- Role: VCMA System
- Requirement ID: NF-05
- Requirement Title: Security
- Requirement Description: The system must provide security features including login method (text password, email, activation code, and anonymously), verify the user’s identity by sending a verification code to the mobile phone, possibility of deleting consultation records, documents, and bank transactions, encrypted patient data exchange.
- Citation: Zahra Parsaei et al., 2024, Table 2
- Requirement Related Title: Nonfunctional Requirements
