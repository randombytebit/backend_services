Functional Requirement  
Role: System  
Requirement ID: FR-01  
Title: Demographic Data Documentation  
Description: System must document patient identity including first name and surname, and insurance details including insurance name and number.  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Page 4, Table 1 (A. Demographic data documentation).  
Related Title: Patient Onboarding in Telemedicine Platforms, Virtual Health Registration Modules  

Functional Requirement  
Role: System  
Requirement ID: FR-02  
Title: Health Record Management  
Description: System must manage comprehensive health records including vital signs (blood pressure, heart rate), kidney documentation (blood sugar), fitness (weight, exercise, sleep, nutrition), skin condition, history (disease, family, allergy, medications, vaccination, surgeries, smoking), present history (symptoms, conditions, X-ray, tests, treatments, prescriptions, patient training), inpatient treatment history (admission/discharge dates, hospital name, chief complaint, summary), and medication details (name, dose, type, instruction, side effects).  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Page 4, Table 1 (B. Health record).  
Related Title: Electronic Health Records in Remote Monitoring Apps, Chronic Patient Data Tracking Systems  

Functional Requirement  
Role: System  
Requirement ID: FR-03  
Title: General Features of User Interface  
Description: System must provide UI features including reminders (appointment, preventive care), display of patient's self-declaration on health status, consultant identity (name, surname, profile picture), curriculum vitae (expertise, consulting field), type of communication (phone call, text, video call, real-time), appointment details (date, time, duration, waiting time), service selection (consultation day/time, insurance type), reporting (weekly/monthly on treatment process, vital signs, medical history), support (communication documentation with tagging and FAQs, Q&A discussion forum), and help/tutorial (text guide, in-app training, about us, terms and conditions).  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Page 4, Table 1 (C. General features of UI).  
Related Title: Teleconsultation Scheduling Interfaces, Virtual Care Patient Portals  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Usability  
Description: System must support usability through settings like creating personal profiles for consultants and patients, microphone and camera access, measurement units selection (e.g., centimeters/meters for height), video calls with other participants, selecting conversation starters, and log out functionality.  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Page 5, Table 2 (A. Usability).  
Related Title: User-Friendly Interfaces in mHealth Apps, Virtual Consultation Setup Tools  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-02  
Title: Accessibility  
Description: System must enable accessibility services such as save and mark functions for appointments, reminders, notes, recorded voices, photos, messages, feedback on consultant performance, and text selection from default lists; and sharing capabilities for individual lessons, health record information with doctors, and patient-consultant data exchange.  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Page 5, Table 2 (B. Accessibility).  
Related Title: Data Sharing in Remote Healthcare Platforms, Accessible Telemedicine Features for Diverse Users

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-05  
Title: Security  
Description: System must ensure security via login methods (text password, email, activation code, anonymously), user identity verification by sending a code to the mobile phone, possibility of deleting consultation records/documents/bank transactions, and encrypted patient data exchange.  
Citation: Parsaei et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Page 5, Table 2 (E. Security).  
Related Title: Secure Data Handling in Telemedicine Apps, Privacy Protection for Virtual Patient Consultations
