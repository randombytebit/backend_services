Functional Requirement
Role: System
Requirement ID: FR-01
Title: Demographic Data Documentation
Description: The system must allow documentation of user identity including first name and surname, as well as insurance details such as insurance name and number.
Citation: Parsaei, Z. et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Table 1, Section A.
Related Title: Patient Registration Systems, Medical Intake Forms

Functional Requirement
Role: System
Requirement ID: FR-02
Title: Health Record Management
Description: The system must enable documentation and management of health records, including vital signs (blood pressure, heart rate), kidney-related data (blood sugar), fitness data (weight, exercise, sleep, nutrition), skin condition, historical data (disease history, family history, allergy, medications history, vaccination, surgeries, smoking), present history (symptoms and conditions, X-ray, tests, treatments, prescriptions, patient training), inpatient treatment history (date of admission and discharge, hospital name, chief complaint, summary), and medication details (name, dose, type, instruction, side effects).
Citation: Parsaei, Z. et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Table 1, Section B.
Related Title: Electronic Health Record Systems, Chronic Disease Monitoring Apps

Functional Requirement
Role: System
Requirement ID: FR-03
Title: General Features of User Interface
Description: The system must provide user interface features including reminders for appointments and preventive care, display of patient self-declaration about health status, consultant identity (name, surname, profile picture), curriculum vitae (expertise and consulting field), type of communication (phone call, text, video call, real-time), appointment details (consultation date, time, duration, waiting time for response), service selection (choosing consultation day, time, and type of insurance), reporting (weekly/monthly reports on treatment process and vital signs/medical history), support (documentation of communication ways including tagging and FAQs, Q&A ability with discussion forums for patients and consultants), and help/tutorial (text guide, in-app training, about us, terms and conditions).
Citation: Parsaei, Z. et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Table 1, Section C.
Related Title: Telemedicine User Interfaces, Healthcare Appointment Schedulers

Non-Functional Requirement
Role: System
Requirement ID: NFR-01
Title: Usability
Description: The system must support usability features such as creating personal profiles for users (consultants and patients), microphone and camera access, selection of measurement units (e.g., centimeters and meters for height), video calls with other participants, selecting conversation starters, and log out functionality.
Citation: Parsaei, Z. et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Table 2, Section A.
Related Title: User-Friendly Mobile Health Apps, Intuitive Teleconsultation Platforms

Non-Functional Requirement
Role: System
Requirement ID: NFR-02
Title: Accessibility
Description: The system must ensure accessibility through services allowing users to save and mark appointments, reminders, notes, recorded voices, photos, and messages; send feedback on consultant performance; select text from default lists; and share individual lessons, health record information with doctors, and patient-consultant data exchange.
Citation: Parsaei, Z. et al. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. Table 2, Section B.
Related Title: Inclusive Healthcare Apps, Accessible Patient Portals

Role: System
Requirement ID: NFR-05
Title: Security
Description: The system must ensure security through login methods (text password, email, activation code, anonymously), user identity verification via verification code sent to mobile phone, possibility of deleting consultation records, documents, and bank transactions, and encrypted patient data exchange.
