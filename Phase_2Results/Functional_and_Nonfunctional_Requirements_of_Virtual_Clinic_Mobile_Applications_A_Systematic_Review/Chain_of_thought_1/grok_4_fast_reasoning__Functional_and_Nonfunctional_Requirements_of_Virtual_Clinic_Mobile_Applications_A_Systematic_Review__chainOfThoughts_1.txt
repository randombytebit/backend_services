### Functional Requirements

**Functional Requirement**  
**Role: System (VCMA Application)**  
**Requirement ID: FR-01**  
**Title: Demographic Data Documentation**  
**Description: The system must allow documentation of user identity including first name and surname, and insurance details including insurance name and number to facilitate patient identification and billing processes.**  
**Citation: Table 1, Section 3.3 (Parsaei et al., 2024).**  
**Related Title: Patient Registration in Hospital Management Systems, Insurance Verification in Telehealth Platforms, User Onboarding in Electronic Health Records**

### Non-Functional Requirements

**Non-Functional Requirement**  
**Role: System (VCMA Application)**  
**Requirement ID: NFR-01**  
**Title: Usability Settings**  
**Description: The system must support usability features including creating personal profiles for consultants and patients, microphone and camera access, selection of measurement units (e.g., centimeters and meters for height), video calls with other participants, selecting conversation starters, and log out functionality to ensure intuitive and user-friendly interactions.**  
**Citation: Table 2, Section 3.3 (Parsaei et al., 2024).**  
**Related Title: Profile Management in Social Health Networks, Multimedia Settings in Video Telehealth, Unit Customization in Global Fitness Trackers**

**Non-Functional Requirement**  
**Role: System (VCMA Application)**  
**Requirement ID: NFR-02**  
**Title: Accessibility Services**  
**Description: The system must provide accessibility through saving and marking appointments, reminders, notes, recorded voices, photos, and messages; sending feedback on consultant performance; and selecting text from a default list, as well as sharing individual lessons, health record information with doctors, and enabling patient-consultant data exchange to promote inclusive access.**  
**Citation: Table 2, Section 3.3 (Parsaei et al., 2024).**  
**Related Title: Note-Sharing in Collaborative Care Platforms, Feedback Mechanisms in Patient Satisfaction Surveys, Data Exchange in Interoperable Health Systems**

**Non-Functional Requirement**  
**Role: System (VCMA Application)**  
**Requirement ID: NFR-03**  
**Title: Compatibility with Insurance Formats**  
**Description: The system must ensure compatibility by recording insurance name and number in service formats to integrate seamlessly with external billing and coverage systems.**  
**Citation: Table 2, Section 3.3 (Parsaei et al., 2024).**  
**Related Title: Insurance Integration in Billing Software, Cross-System Compatibility in Payer Portals, Standardized Data Formats in Healthcare APIs**

**Non-Functional Requirement**  
**Role: System (VCMA Application)**  
**Requirement ID: NFR-04**  
**Title: Efficiency in Financial Transactions**  
**Description: The system must handle efficiency through financial transaction reporting to ensure timely and accurate processing of payments and reimbursements.**  
**Citation: Table 2, Section 3.3 (Parsaei et al., 2024).**  
**Related Title: Transaction Logging in Medical Billing Systems, Payment Efficiency in E-Commerce Health Services, Reporting Tools in Financial Health Analytics**

**Non-Functional Requirement**  
**Role: System (VCMA Application)**  
**Requirement ID: NFR-05**  
**Title: Security Measures**  
**Description: The system must implement security via login methods (text password, email, activation code, anonymously), user identity verification by sending codes to mobile phones, possibility of deleting consultation records/documents/bank transactions, and encrypted patient data exchange to protect privacy and data integrity.**  
**Citation: Table 2, Section 3.3 (Parsaei et al., 2024).**  
**Related Title: Authentication in HIPAA-Compliant EHRs, Data Encryption in Telemedicine Security Protocols, Record Deletion in GDPR-Compliant Health Apps**
