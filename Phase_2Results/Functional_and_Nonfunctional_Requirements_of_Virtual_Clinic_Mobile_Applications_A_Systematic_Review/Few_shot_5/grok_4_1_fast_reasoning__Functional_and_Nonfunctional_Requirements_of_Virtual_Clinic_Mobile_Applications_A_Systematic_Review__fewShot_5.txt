Functional Requirement  
Role: System  
Requirement ID: FR-01  
Title: Demographic Data Documentation  
Description: System should document demographic data including identity (first name and surname) and insurance (insurance name and number).

Functional Requirement  
Role: System  
Requirement ID: FR-02  
Title: Health Record Management  
Description: System should manage health records including vital signs documentation (blood pressure, heart rate), kidney documentation (blood sugar), fitness documentation (weight, exercise, sleep, nutrition), skin documentation (condition of skin), history documentation (disease history, family history, allergy, medications history, vaccination, surgeries, smoking), present history documentation (symptoms and conditions, X-ray, tests, treatments, prescriptions, patient training), documentation of inpatient treatment history (date of admission and discharge, hospital name, chief complaint, summary), and medication documentation (name, dose, type, instruction, side effects).  
Citation: Page 4, Table 1  
Related Title: Electronic Health Records in Telemedicine, Patient Vital Signs Tracking in mHealth  

Functional Requirement  
Role: System  
Requirement ID: FR-03  
Title: General Features of User Interface  
Description: System should provide general UI features including reminders (appointment, preventive care), display (patient’s self-declaration about health status, consultant identity with name/surname/profile picture, curriculum vitae with expertise/consulting field, type of communication such as phone call/text/video call/real-time, appointment details like date/time/duration/waiting time), service selection (choosing consultation day/time/type of insurance), reporting (weekly/monthly reports on patient’s treatment process, weekly/monthly reports of vital signs and medical history), support (documentation of communication ways with tagging capability and FAQs, Q&A ability via discussion forum with patients/consultants), and help/tutorial (text guide/help, training inside VCMA on usage, about us, terms and conditions of use).  
Citation: Page 4, Table 1  
Related Title: UI Reminders and Reporting in Virtual Clinics, Telehealth Consultation Scheduling Interfaces

Title: Usability  
Description: System should support usability features including settings for creating personal profile (consultant-patient), microphone, camera, measurement units selection (e.g., centimeters/meters for height), video call with other participants, selecting conversation starter, and log out.
