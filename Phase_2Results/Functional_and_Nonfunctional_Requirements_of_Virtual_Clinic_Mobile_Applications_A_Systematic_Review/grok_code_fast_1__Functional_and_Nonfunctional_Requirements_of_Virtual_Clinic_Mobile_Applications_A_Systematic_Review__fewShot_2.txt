Functional Requirement  
Role: User (Patient)  
Requirement ID: FR-01  
Title: Demographic Data Documentation  
Description: The system must enable users to input and store personal identity information including first name and surname, as well as insurance details such as insurance name and number, to facilitate patient identification and billing processes in virtual consultations.

Functional Requirement  
Role: User (Patient)  
Requirement ID: FR-02  
Title: Health Record Management  
Description: The system must support comprehensive documentation and management of health records, including vital signs (e.g., blood pressure, heart rate), kidney-related data (e.g., blood sugar), fitness metrics (e.g., weight, exercise, sleep, nutrition), skin conditions, historical data (e.g., disease history, family history, allergies, medications, vaccinations, surgeries, smoking), present conditions (e.g., symptoms, X-rays, tests, treatments, prescriptions, patient training), inpatient treatment history (e.g., admission/discharge dates, hospital name, chief complaint, summary), and medication details (e.g., name, dose, type, instructions, side effects), to enable clinicians to access accurate patient information for diagnosis and treatment planning.  
Citation: Page 4, Table 1, B. Health record  
Related Title: Electronic Health Record (EHR) Integration in Mobile Health Apps  

Functional Requirement  
Role: User (Patient or Clinician)  
Requirement ID: FR-03  
Title: User Interface General Features  
Description: The system must provide a user-friendly interface with features such as reminders for appointments and preventive care, display options for patient self-declarations on health status, consultant profiles (name, surname, picture, CV, expertise), communication types (phone, text, video calls, real-time), appointment scheduling (date, time, duration, waiting time), service selection (consultation day, time, insurance type), reporting (weekly/monthly treatment and vital signs reports), support (communication documentation, tagging, FAQs, Q&A forums), and help/tutorial (text guides, in-app training, about us, terms and conditions), to enhance usability and facilitate virtual clinic interactions.  
Citation: Page 4, Table 1, C. General features of UI
