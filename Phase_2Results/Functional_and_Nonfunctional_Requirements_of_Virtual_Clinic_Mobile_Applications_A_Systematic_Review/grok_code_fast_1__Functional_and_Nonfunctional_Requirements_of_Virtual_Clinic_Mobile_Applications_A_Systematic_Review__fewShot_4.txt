Functional Requirement
Role: User
Requirement ID: FR-01
Title: Demographic Data Documentation
Description: The system must allow users to document and manage demographic information, including identity details such as first name and surname, as well as insurance information like name and number, to facilitate personalized clinical services.
Citation: Parsaei, Z., Jangi, M., Tahmasebian, S., & Ehteshami, A. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. International Journal of Telemedicine and Applications, 2024.

Functional Requirement
Role: User
Requirement ID: FR-02
Title: Health Record Management
Description: The system must enable users to document and access comprehensive health records, including vital signs (e.g., blood pressure, heart rate), kidney documentation (e.g., blood sugar), fitness documentation (e.g., weight, exercise, sleep, nutrition), skin documentation (e.g., condition), history documentation (e.g., disease history, family history, allergy, medications, vaccination, surgeries, smoking), present history documentation (e.g., symptoms, conditions, X-ray, tests, treatments, prescriptions, patient training), inpatient treatment history (e.g., date of admission and discharge, hospital name, chief complaint, summary), and medication documentation (e.g., name, dose, type, instruction, side effects), to support clinical decision-making and patient self-management.
Citation: Parsaei, Z., Jangi, M., Tahmasebian, S., & Ehteshami, A. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. International Journal of Telemedicine and Applications, 2024.
Related Title: Electronic Health Record Systems in Virtual Care Environments

Functional Requirement
Role: User
Requirement ID: FR-03
Title: General Features of User Interface
Description: The system must provide general UI features to enhance user interaction, including reminders (e.g., for appointments and preventive care), display options (e.g., patient's self-declaration about health status, consultant identity with name, surname, profile picture, curriculum vitae, expertise, consulting field, type of communication such as phone call, text, video call, real-time, appointment details like date, time, duration, waiting time), service selection (e.g., choosing consultation day, time, and type of insurance), reporting (e.g., weekly/monthly reports on treatment process, vital signs, and medical history), support (e.g., documentation of communication ways, tagging capability, frequently asked questions, Q and A ability, discussion forum), and help/tutorial (e.g., text guide, training inside the app, about us, terms and conditions).
Citation: Parsaei, Z., Jangi, M., Tahmasebian, S., & Ehteshami, A. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. International Journal of Telemedicine and Applications, 2024.
Related Title: User Interface Modules in Mobile Health Applications

Non-Functional Requirement
Role: System
Requirement ID: NFR-01
Title: Usability
Description: The system must ensure usability through settings that allow creating personal profiles for users (consultants and patients), configuring microphone and camera, selecting measurement units (e.g., centimeters and meters for height), enabling video calls with other participants, selecting conversation starters, and providing logout functionality, to facilitate intuitive and efficient user interactions.
Citation: Parsaei, Z., Jangi, M., Tahmasebian, S., & Ehteshami, A. (2024). Functional and Nonfunctional Requirements of Virtual Clinic Mobile Applications: A Systematic Review. International Journal of Telemedicine and Applications, 2024.

Role: System
Requirement ID: NFR-02
Title: Accessibility
Description: The system must support accessibility features such as saving and marking appointments, reminders, notes, recorded voices, photos, and messages; sending feedback on consultant performance; selecting text from a default list; and sharing individual lessons, health record information with doctors, and patient-consultant data exchange, to ensure broad usability across diverse user needs.

Non-Functional Requirement
Role: System
Requirement ID: NFR-05
Title: Security
Description: The system must enforce security measures including login methods (e.g., text password, email, activation code, anonymously), user identity verification via verification code sent to mobile phone, possibility of deleting consultation records, documents, and bank transactions, and encrypted patient data exchange, to protect sensitive health information and prevent unauthorized access.
