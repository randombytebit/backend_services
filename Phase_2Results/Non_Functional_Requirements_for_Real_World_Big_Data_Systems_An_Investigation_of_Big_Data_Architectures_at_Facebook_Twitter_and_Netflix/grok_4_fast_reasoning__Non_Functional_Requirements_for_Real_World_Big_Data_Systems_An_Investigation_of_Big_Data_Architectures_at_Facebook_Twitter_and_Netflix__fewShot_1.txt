Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Batch Data Processing  
Description: The system must support processing of finite, large-volume data stored in distributed file systems or databases, such as hundreds of petabytes on disk with daily processing volumes in the tens of petabytes, using integrated storage and query systems like Operational Data Stores (ODS), Scuba, Hive, Laser, Hadoop clusters, Scalding, Presto, or cloud-based installations with Genie and Amazon S3.  
Citation: Section 3.1 Batch Data  
Related Title: Social Media Data Warehousing, E-commerce Analytics Platforms, Enterprise Data Lakes  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-02  
Title: Stream Data Processing  
Description: The system must handle potentially infinite, high-velocity data streams in real-time or near real-time, such as monitoring data, using specialized frameworks like Puma, Swift, and Stylus for SQL-like queries and DAG-based processing; Storm/Heron with Mesos and Aurora for cluster management; or Mantis with Fenzo for scheduling on cloud resources.  
Citation: Section 3.2 Stream Data  
Related Title: Real-Time Fraud Detection Systems, IoT Sensor Networks, Live Event Analytics  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-03  
Title: Late and Out-of-Order Data Handling  
Description: In stream processing, the system must manage data that arrives late or out of sequence (e.g., due to network delays in mobile streams) by using mechanisms like data windows, low watermarks in Stylus, or lag specifications in BaseWindowedBolt to process finite windows of infinite streams without data loss or corruption.  
Citation: Section 3.3 Late and Out of Order Data  
Related Title: Mobile App Telemetry Processing, Financial Trading Feeds, Video Streaming Buffering Systems  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-04  
Title: Processing Guarantees  
Description: The system must provide configurable processing semantics for streams, including exactly-once (via checkpointing, idempotency, and atomic transactions in Kafka), at-least-once (with optimizations in Stylus and Heron), or at-most-once (for sampling-tolerant systems like Scuba), balancing completeness, duplication avoidance, and latency costs.  
Citation: Section 3.4 Processing Guarantees  
Related Title: Transactional Banking Systems, Log Aggregation Pipelines, Content Recommendation Engines  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-05  
Title: Integration and Extensibility  
Description: The system must integrate with existing services via standardized messaging (e.g., Scribe for pipelines across Puma, Swift, and Stylus) and support extensibility through modular designs (e.g., decoupled components in Heron for adaptation to new infrastructures) or single-framework orchestration (e.g., Titus on Mesos for efficient resource visibility), enabling seamless addition of future components.  
Citation: Section 3.5 Integration and Extensibility  
Related Title: Microservices Orchestration Platforms, API Gateway Integrations, Hybrid Cloud Data Pipelines  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-06  
Title: Distribution and Scalability  
Description: The system must distribute processing across multi-clustered machines in different data centers, supporting dynamic scaling without downtime by adding/removing nodes (e.g., scaling Scribe brokers for parallelism, using Dhalion in Heron for cluster adjustment, or Fenzo in Mesos for EC2 node provisioning), handling petabyte-scale workloads.  
Citation: Section 3.6 Distribution and Scalability  
Related Title: Distributed Computing Grids, Global Content Delivery Networks, High-Throughput Genomics Pipelines  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-07  
Title: Cloud Support and Elasticity  
Description: The system must deploy to cloud environments for elasticity, scaling resources up/down based on demand to minimize costs and wastage (e.g., full cloud-based Hadoop with Mesos/Fenzo on AWS EC2 for Netflix; containerized shifts in Twitter's Heron; potential multi-cloud migrations via containers to avoid vendor lock-in).  
Citation: Section 3.7 Cloud Support and Elasticity  
Related Title: Serverless Data Processing Platforms, Auto-Scaling Web Analytics, Multi-Cloud Backup Systems  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-08  
Title: Fault Tolerance  
Description: The system must continue operating despite node failures, recovering gracefully with minimal user impact through node independence, persistent messaging (e.g., Scribe backed by Swift checkpointing at Facebook), modular/distributed designs (e.g., decentralized topology masters and JVM isolation in Heron), or regional redundancy (e.g., Active-Active deployments across AWS regions like US-East-1, US-West-2, and EU-West-1 at Netflix).  
Citation: Section 3.8 Fault Tolerance  
Related Title: High-Availability E-Commerce Platforms, Disaster Recovery Systems, Autonomous Vehicle Data Processing  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-09  
Title: Flow Control  
Description: The system must manage backpressure when data emission exceeds consumption rates, using strategies like node reallocation via persistent messaging (Scribe at Facebook), upstream throttling in topologies (Heron tightening spouts to match slowest components), or programmatic options (RxJava in Mantis for buffering, sampling, debouncing, or windowing hot observables).  
Citation: Section 3.9 Flow Control  
Related Title: Real-Time Sensor Data Ingestion, Network Traffic Monitoring, Overloaded Queue Management Systems  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-10  
Title: Flexibility and Technology Agnosticism  
Description: The system must allow replacement of components with alternative technologies without major redesign, through modular architectures (e.g., Heron's decoupling for Kubernetes or Microsoft adoption), standards like Reactive Streams for interoperability (Netflix's Mantis with RxJava and JDK 9), container-based deployments to reduce infrastructure coupling (Netflix's Titus shift), and open-sourcing for community extensions.  
Citation: Section 3.10 Flexibility and Technology Agnosticism  
Related Title: Open-Source Data Framework Ecosystems, Vendor-Neutral Analytics Tools, Portable ML Model Serving Platforms
