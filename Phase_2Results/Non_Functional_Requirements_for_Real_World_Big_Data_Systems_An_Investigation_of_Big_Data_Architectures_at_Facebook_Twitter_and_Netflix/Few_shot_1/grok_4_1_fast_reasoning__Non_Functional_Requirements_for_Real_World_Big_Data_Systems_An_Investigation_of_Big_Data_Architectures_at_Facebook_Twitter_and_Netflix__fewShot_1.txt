Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Batch Data Processing  
Description: System must support processing of finite, large-volume data (e.g., hundreds of petabytes stored on disk, tens of petabytes processed daily) archived in distributed file systems or databases, using tools like Operational Data Store (ODS), Scuba, Hive, Laser (Facebook), Hadoop/Scalding/Presto (Twitter), or cloud-based Hadoop/Genie/S3 (Netflix).  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.1 Batch Data (Pages 3-4)  
Related Title: Social Media Data Warehouses, ETL Pipelines for Enterprise Analytics, Log Aggregation Systems  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-02  
Title: Stream Data Processing  
Description: System must support processing of potentially infinite, high-velocity streaming data (e.g., real-time or near real-time monitoring), using frameworks like Puma/Swift/Stylus (Facebook), Storm/Heron with Mesos/Aurora (Twitter), or Mantis with Mesos/Fenzo (Netflix).  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.2 Stream Data (Page 4)  
Related Title: Real-Time Fraud Detection Platforms, IoT Sensor Stream Processors, Live Video Analytics Systems  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-03  
Title: Late and Out-of-Order Data Handling  
Description: System must handle late or out-of-order streaming data (e.g., delayed mobile user streams) using mechanisms like data windows, low watermarks (Facebook Stylus), withLag() in BaseWindowedBolt (Twitter Heron), transforming infinite streams into finite processable windows.  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.3 Late and Out of Order Data (Page 4)  
Related Title: Mobile Network Telemetry Systems, Financial Tick Data Processors, GPS Tracking Analytics  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-04  
Title: Processing Guarantees  
Description: System must provide configurable processing semantics including exactly-once (via checkpointing/idempotency/atomic transactions, e.g., Netflix Kafka), at-least-once (e.g., Facebook Stylus/Swift), at-most-once (e.g., Facebook Scuba), and effectively-once (Twitter Heron), balancing completeness, duplication, and latency.  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.4 Processing Guarantees (Page 4)  
Related Title: Transactional Stream Databases, Event Sourcing Systems, Reliable Message Queues  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-05  
Title: Integration and Extensibility  
Description: System must integrate with existing services/components via shared messaging (e.g., Facebook Scribe), support modular decoupling for extensions (e.g., Twitter Heron), and allow seamless pipeline creation/orchestration (e.g., Netflix Titus/Mesos), facilitating future component incorporation.  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.5 Integration and Extensibility (Pages 4-5)  
Related Title: Microservices Orchestration Platforms, Plugin-Based Data Pipelines, API Gateway Integrations  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-06  
Title: Distribution and Scalability  
Description: System must distribute processing across multi-machine/multi-data-center clusters with dynamic scaling (add/remove nodes without downtime), e.g., via scalable brokers (Facebook Scribe), Dhalion auto-scaling (Twitter Heron), or EC2 node adjustments (Netflix Fenzo/Mesos).  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.6 Distribution and Scalability (Page 5)  
Related Title: Multi-Region CDN Analytics, Distributed Machine Learning Clusters, Global E-Commerce Recommendation Engines  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-07  
Title: Cloud Support and Elasticity  
Description: System must support cloud deployment for elasticity (scale up/down by demand, pay-per-use), e.g., full AWS shift with containers (Netflix), containerized non-cloud setups migratable to multi-cloud (Twitter Heron), combining scalability with cloud economies over fixed infrastructure.  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.7 Cloud Support and Elasticity (Page 5)  
Related Title: Serverless Data Processing Platforms, Auto-Scaling Kubernetes Clusters, Multi-Cloud Hybrid Analytics  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-08  
Title: Fault Tolerance  
Description: System must continue operation despite node failures with graceful recovery, via node independence/persistent messaging/checkpointing (Facebook Scribe/Swift), modular/container/JVM/topology isolation (Twitter Heron), or multi-region Active-Active redundancy (Netflix AWS regions).  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.8 Fault Tolerance (Pages 5-6)  
Related Title: High-Availability Banking Systems, Disaster Recovery Streaming Platforms, Geo-Redundant Cloud Services  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-09  
Title: Flow Control and Backpressure  
Description: System must manage backpressure (data emitted faster than consumed) via strategies like node reallocation (Facebook Scribe), upstream tightening/slowest-component pacing (Twitter Heron), or programmatic buffering/sampling/debouncing/windowing (Netflix Mantis/RxJava).  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.9 Flow Control (Page 6)  
Related Title: High-Throughput Log Processors, Reactive Web Socket Handlers, Overloaded Sensor Networks  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-10  
Title: Flexibility and Technology Agnosticism  
Description: System must support interchangeable technologies/components (e.g., non-compulsory Mesos/Aurora in Twitter Heron, Reactive Streams/RxJava in Netflix Mantis, container decoupling from VMs), with modular/open-source designs reducing vendor lock-in (vs. Facebook's in-house specificity).  
Citation: Vergilio, T and Ramachandran, M (2019), Section 3.10 Flexibility and Technology Agnosticism (Pages 6-7)  
Related Title: Framework-Agnostic Stream Processors, Open-Source Data Pipeline Toolkits, Kubernetes-Native Big Data Deployments
