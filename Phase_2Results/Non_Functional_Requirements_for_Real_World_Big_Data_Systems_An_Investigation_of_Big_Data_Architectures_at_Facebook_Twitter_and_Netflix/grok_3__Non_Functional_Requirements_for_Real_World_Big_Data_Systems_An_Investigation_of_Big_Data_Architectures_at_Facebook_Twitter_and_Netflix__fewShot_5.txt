**Non-Functional Requirement**  
**Role:** System  
**Requirement ID:** NFR-01  
**Title:** Batch Data Processing Capability  
**Description:** The system must have the capability to process finite, large-volume data, such as data archived in distributed file systems or databases. This includes handling hundreds of petabytes of stored data and processing tens of petabytes daily, as exemplified by architectures at Facebook, Twitter, and Netflix.  
**Citation:** Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1, pp. 833-840. [Section 3.1]  
**Related Title:** Large-Scale Data Archiving Systems

**Non-Functional Requirement**  
**Role:** System  
**Requirement ID:** NFR-02  
**Title:** Stream Data Processing Capability  
**Description:** The system must be capable of processing potentially infinite data flowing at high velocity, such as real-time or near real-time monitoring data. This includes using specialized frameworks like Puma, Swift, and Stylus at Facebook, Storm and Heron at Twitter, and Mantis at Netflix.  
**Citation:** Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1, pp. 833-840. [Section 3.2]

**Non-Functional Requirement**  
**Role:** System  
**Requirement ID:** NFR-04  
**Title:** Processing Guarantees  
**Description:** The system must offer processing guarantees for stream data, including exactly-once, at-least-once, and at-most-once semantics, balancing latency costs with data completeness requirements as implemented by Facebook (Scuba, Stylus), Twitter (Heron), and Netflix (Kafka).  
**Citation:** Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. **Non-Functional Requirement**  
**Role:** System  
**Requirement ID:** NFR-08  
**Title:** Fault Tolerance
