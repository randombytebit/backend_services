Non-Functional Requirement
Role: System
Requirement ID: NFR-01
Title: Batch Data
Description: The capability to process data which is finite and usually large in volume, e.g. data archived in distributed file systems or databases, with examples including hundreds of petabytes stored on disk and daily processing volumes in the tens of petabytes for companies like Facebook and Twitter, and tens of petabytes stored with approximately 3 petabytes daily reads for Netflix.
Citation: Vergilio & Ramachandran (2019), Section 3.1
Related Title: Social Media Data Warehousing, Financial Transaction Archiving, E-commerce Sales Analytics

Non-Functional Requirement
Role: System
Requirement ID: NFR-02
Title: Stream Data
Description: The capability to process data which is potentially infinite and usually flowing at high velocity, e.g. monitoring data captured and processed in real-time or close to real-time, implemented through in-house systems like Puma, Swift, and Stylus at Facebook, Storm and Heron at Twitter, and Mantis at Netflix.
Citation: Vergilio & Ramachandran (2019), Section 3.2
Related Title: Real-Time Social Media Monitoring, Live Video Streaming Analytics, IoT Sensor Data Processing

Non-Functional Requirement
Role: System
Requirement ID: NFR-03
Title: Late and Out of Order Data
Description: The capability to process data which arrives late or in a different order from that in which it was emitted, particularly in stream processing, using mechanisms like windows of data and low watermarks in Stylus at Facebook, withLag() method in Heron's BaseWindowedBolt at Twitter, and unspecified strategies in Mantis at Netflix.
Citation: Vergilio & Ramachandran (2019), Section 3.3
Related Title: Mobile User Event Tracking, Financial Market Data Feeds, Supply Chain Logistics Monitoring

Non-Functional Requirement
Role: System
Requirement ID: NFR-04
Title: Processing Guarantees
Description: The capability to offer processing guarantees in stream systems, such as exactly once, at least once, and at most once, with implementations including at least once in Stylus at Facebook, at least once and at most once in Storm and Heron at Twitter (with effectively once in Heron), and exactly once via Kafka at Netflix.
Citation: Vergilio & Ramachandran (2019), Section 3.4
Related Title: Online Banking Transaction Processing, E-commerce Order Fulfillment, Healthcare Patient Monitoring

Non-Functional Requirement
Role: System
Requirement ID: NFR-05
Title: Integration and Extensibility
Description: The capability to integrate with existing services and components, and to facilitate extension of the architecture to incorporate different components in the future, achieved through messaging systems like Scribe at Facebook, modular architecture in Heron at Twitter, and rigid but efficient single-framework designs like Titus on Mesos at Netflix.
Citation: Vergilio & Ramachandran (2019), Section 3.5
Related Title: Enterprise Service Bus Integration, Cloud Microservices Expansion, API Gateway Extensions

Non-Functional Requirement
Role: System
Requirement ID: NFR-06
Title: Distribution and Scalability
Description: The capability to distribute data processing amongst different machines in multi-clustered architectures, including dynamic scaling without downtime, implemented via scalable messaging like Scribe at Facebook, Dhalion framework in Heron at Twitter, and elastic resource management via Fenzo at Netflix.
Citation: Vergilio & Ramachandran (2019), Section 3.6
Related Title: Global Web Traffic Distribution, High-Volume E-commerce Scaling, Distributed Machine Learning Training

Non-Functional Requirement
Role: System
Requirement ID: NFR-07
Title: Cloud Support and Elasticity
Description: The capability to move the architecture into the cloud for benefits like economies of scale and elasticity, allowing scaling up and down according to demand, with Netflix's architecture being predominantly cloud-based on AWS, Twitter shifting towards containerized Heron, and Facebook's systems less cloud-oriented.
Citation: Vergilio & Ramachandran (2019), Section 3.7
Related Title: On-Demand Video Streaming Platforms, Seasonal Retail Analytics, Dynamic Ad Serving Networks

Non-Functional Requirement
Role: System
Requirement ID: NFR-08
Title: Fault Tolerance
Description: The capability of a system to continue operating should one or more nodes fail, recovering gracefully with minimal impact, implemented through node independence and checkpointing via Scribe and Swift at Facebook, modular distributed architecture and decentralized topology management in Heron at Twitter, and multi-region redundancy in Active-Active deployments at Netflix.
Citation: Vergilio & Ramachandran (2019), Section 3.8
Related Title: Critical Infrastructure Monitoring, Autonomous Vehicle Control Systems, Financial Trading Platforms

Non-Functional Requirement
Role: System
Requirement ID: NFR-09
Title: Flow Control
Description: The capability to handle scenarios where the data source emits records faster than the system can consume, using strategies like backpressure management, such as resilient pipelines via Scribe at Facebook, tightening upstream components in Heron at Twitter, and programmatic methods via ReactiveX libraries in Mantis at Netflix.
Citation: Vergilio & Ramachandran (2019), Section 3.9
Related Title: High-Frequency Trading Systems, Real-Time Gaming Servers, Live Event Broadcasting

Non-Functional Requirement
Role: System
Requirement ID: NFR-10
Title: Flexibility and Technology Agnosticism
Description: The capability of an architecture to use different technologies in place of existing components, with varying degrees of flexibility, such as Facebook's in-house systems being less agnostic, Heron's modular design allowing adoption on different infrastructures like Kubernetes, and Netflix's use of containers and reactive standards promoting interoperability.
Citation: Vergilio & Ramachandran (2019), Section 3.10
Related Title: Multi-Vendor Cloud Deployments, Open-Source Framework Adoption, Cross-Platform Mobile App Development
