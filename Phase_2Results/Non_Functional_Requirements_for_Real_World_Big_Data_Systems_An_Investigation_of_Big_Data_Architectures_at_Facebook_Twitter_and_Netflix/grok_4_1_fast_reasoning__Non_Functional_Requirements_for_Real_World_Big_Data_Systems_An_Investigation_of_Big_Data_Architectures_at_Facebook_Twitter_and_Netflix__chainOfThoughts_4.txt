Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Processing Guarantees  
Description: System should offer processing guarantees (i.e., exactly once, at least once, and at most once) in stream processing, balancing reliability with latency costs like checkpointing.  
Citation: Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, pp. 833-840.[cite:1]

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-02  
Title: Fault Tolerance  
Description: System should continue to operate if one or more nodes fail, recovering gracefully with minimal impact on user experience, e.g., via node independence, checkpointing, and redundancy.  
Citation: Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, pp. 833-840.[cite:1]
