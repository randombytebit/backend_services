Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-01  
Title: Batch Data Processing  
Description: The system must have the capability to process finite and usually large volumes of data, such as data archived in distributed file systems or databases, with examples including hundreds of petabytes stored on disk and tens of petabytes processed daily at companies like Facebook and Twitter, or tens of petabytes stored and 3 petabytes read daily at Netflix.  
Citation: Section 3.1, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: Social Media Data Archiving System, Enterprise Data Warehousing System, Log Analysis System  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-02  
Title: Stream Data Processing  
Description: The system must have the capability to process potentially infinite data flowing at high velocity, such as real-time or near real-time monitoring data, using specialized stream processing frameworks like Puma, Swift, and Stylus at Facebook; Heron (replacing Storm) at Twitter; or Mantis at Netflix.  
Citation: Section 3.2, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: Real-Time Fraud Detection System, Live Video Streaming Analytics System, IoT Sensor Data Processing System  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-03  
Title: Handling Late and Out-of-Order Data  
Description: In stream processing, the system must handle data that arrives late or out of the original emission order, such as delayed streaming data from mobile users, by using mechanisms like data windows to process finite segments of infinite streams, low watermarks in Facebook's Stylus, or lag specification methods in Twitter's Heron BaseWindowedBolt class.  
Citation: Section 3.3, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: Mobile Network Traffic Monitoring System, Stock Market Tick Data System, GPS Tracking Analytics System  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-04  
Title: Processing Guarantees  
Description: The stream processing system must provide guarantees such as exactly once, at least once, or at most once semantics to ensure data processing reliability, balancing completeness and latency; for example, Facebook's Stylus offers at least once via checkpointing, Twitter's Heron provides effectively once semantics, and Netflix's Kafka supports exactly once through idempotency and atomic transactions.  
Citation: Section 3.4, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: Financial Transaction Processing System, E-Commerce Order Fulfillment System, Healthcare Patient Record System  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-05  
Title: Integration and Extensibility  
Description: The system must integrate with existing services and components while facilitating future extensions, such as through a central messaging system like Facebook's Scribe for seamless pipelines, Twitter's modular Heron architecture for decoupling and adoption, or Netflix's Mesos-based Titus for efficient resource allocation despite some rigidity.  
Citation: Section 3.5, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: Enterprise Service Bus Integration System, Microservices Orchestration Platform, API Gateway Extension System  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-06  
Title: Distribution and Scalability  
Description: The system must distribute data processing across multiple machines and data centers in a multi-clustered architecture, supporting dynamic scaling without downtime by adding or removing nodes, as seen in Facebook's Scribe for parallelism, Twitter's Dhalion for cluster adjustment in Heron, and Netflix's Fenzo for EC2 node scaling in Mesos.  
Citation: Section 3.6, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: Distributed Cloud Computing Cluster, High-Traffic Web Analytics System, Global Content Delivery Network  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-07  
Title: Cloud Support and Elasticity  
Description: The system must support deployment to the cloud for economies of scale and elasticity, allowing dynamic scaling up or down based on demand to minimize costs and handle spikes, with Netflix's architecture being predominantly cloud-based on AWS using containers, Twitter shifting to containerized setups for multi-cloud potential, and Facebook relying more on in-house scalability.  
Citation: Section 3.7, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: Serverless Computing Platform, Auto-Scaling E-Commerce Backend, Demand-Responsive Video Streaming Service  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-08  
Title: Fault Tolerance  
Description: The system must continue operating despite node failures, recovering gracefully with minimal impact on user experience through mechanisms like node independence and persistent messaging in Facebook's Scribe and Swift, modular distribution and decentralized management in Twitter's Heron, or regional redundancy across AWS regions in Netflix's Active-Active project.  
Citation: Section 3.8, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: High-Availability Banking System, Disaster Recovery Data Center, Mission-Critical IoT Network  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-09  
Title: Flow Control  
Description: The system must handle scenarios where data sources emit records faster than consumption rates, managing backpressure through strategies like node reallocation in Facebook's Scribe-based architecture, upstream tightening in Twitter's Heron to match the slowest component, or programmatic options like buffering and sampling in Netflix's Mantis using ReactiveX libraries.  
Citation: Section 3.9, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: High-Volume Sensor Data Pipeline, Real-Time Ad Serving System, Network Traffic Management System  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-10  
Title: Flexibility and Technology Agnosticism  
Description: The system must allow replacement of components with different technologies for adaptability, with Twitter's Heron offering modular design compatible with Mesos, Aurora, or Kubernetes; Netflix supporting Reactive Streams for interoperability and container-based decoupling from AWS specifics; while Facebook's in-house systems like Scribe are less flexible but partially open-sourced.  
Citation: Section 3.10, Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840.  
Related Title: Hybrid Cloud Migration Framework, Open-Source Framework Adaptation System, Vendor-Neutral Analytics Platform
