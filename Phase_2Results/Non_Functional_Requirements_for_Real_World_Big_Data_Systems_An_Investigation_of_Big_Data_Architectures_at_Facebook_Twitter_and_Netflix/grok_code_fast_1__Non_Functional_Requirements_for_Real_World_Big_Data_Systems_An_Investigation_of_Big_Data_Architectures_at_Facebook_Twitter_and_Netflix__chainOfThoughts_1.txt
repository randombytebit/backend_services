Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-01  
Title: Batch Data Processing  
Description: The system must be capable of processing finite and large volumes of data, such as data archived in distributed file systems or databases, with examples including hundreds of petabytes stored on disk and daily processing volumes of tens of petabytes.  
Citation: Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840  
Related Title: Financial Transaction Ledger Systems, Genomic Sequencing Databases, E-commerce Inventory Management Platforms  

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-02  
Title: Stream Data Processing  
Description: The system must be capable of processing potentially infinite data flowing at high velocity, such as monitoring data captured and processed in real-time or close to real-time.

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-03  
Title: Handling Late and Out-of-Order Data  
Description: The system must be capable of processing streaming data that arrives late or in a different order from emission, such as data from mobile users delayed by reception issues, by transforming infinite streams into finite windows for individual processing.  
Citation: Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840

Non-Functional Requirement  
Role: Big Data System  
Requirement ID: NFR-04  
Title: Processing Guarantees  
Description: The system must provide processing guarantees for stream data, including exactly once, at least once, and at most once semantics, where exactly once involves checkpointing for state persistence but may increase latency, and choices depend on use-cases like sampling where duplication is unacceptable.  
Citation: Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840  
Related Title: Banking Transaction Processing Systems, Online Auction Platforms, Supply Chain Tracking Systems
