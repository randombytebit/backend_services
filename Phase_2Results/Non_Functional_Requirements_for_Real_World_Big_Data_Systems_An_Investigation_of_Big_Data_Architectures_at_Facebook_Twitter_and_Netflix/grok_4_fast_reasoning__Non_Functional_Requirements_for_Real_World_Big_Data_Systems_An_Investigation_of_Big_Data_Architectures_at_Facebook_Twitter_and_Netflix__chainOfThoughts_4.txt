Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Processing Guarantees  
Description: System should provide processing guarantees for stream data, including exactly once, at least once, and at most once semantics, to ensure reliable delivery and minimize duplication or loss during runtime failures, with optimizations to balance latency and completeness (e.g., using checkpointing for exactly once, though it may increase latency).  
Citation: Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, pp. 833-840. [Section 3.4]  
Related Title: Stream Processing Pipelines, Real-Time Analytics Systems  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-02  
Title: Fault Tolerance  
Description: System should continue operating with minimal disruption if one or more nodes fail, achieving graceful recovery through mechanisms such as node independence, persistent messaging for checkpointing, modular distributed architecture, decentralized management, resource isolation, and multi-regional redundancy to isolate failures and restart affected components without impacting overall user experience.  
Citation: Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, pp. 833-840. [Section 3.8]  
Related Title: Distributed Cluster Architectures, Cloud-Native Data Platforms  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-03  
Title: Late and Out of Order Data Handling  
Description: System should process streaming data that arrives late or out of order (e.g., due to network delays) by transforming infinite streams into finite windows and applying mechanisms like low watermarks or lag tolerances to ensure reliable ordering and completeness without data loss during runtime.  
Citation: Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, pp. 833-840. [Section 3.3]  
Related Title: Real-Time Stream Processing Engines, Mobile Data Ingestion Systems  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-04  
Title: Flow Control  
Description: System should manage backpressure scenarios where data ingestion exceeds processing capacity, using strategies such as node reallocation, upstream throttling, buffering, sampling, or reactive observables to prevent pipeline slowdowns or data drops, ensuring runtime stability and reliability under variable loads.  
Citation: Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, pp. 833-840. [Section 3.9]  
Related Title: High-Velocity Data Pipelines, Reactive Streaming Frameworks
