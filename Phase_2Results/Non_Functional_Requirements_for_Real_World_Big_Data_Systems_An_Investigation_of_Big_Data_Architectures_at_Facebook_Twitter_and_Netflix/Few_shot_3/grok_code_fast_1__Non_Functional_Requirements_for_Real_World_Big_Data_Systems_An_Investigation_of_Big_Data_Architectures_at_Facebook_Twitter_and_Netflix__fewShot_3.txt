Non-Functional Requirement
Role: System
Requirement ID: NFR-01
Title: Batch Data
Description: The capability to process data which is finite and usually large in volume, e.g. data archived in distributed file systems or databases, with volumes reaching hundreds of petabytes daily at companies like Facebook and Twitter, and tens of petabytes at Netflix.
Citation: Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1, pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840 [Section 3.1]
Related Title: Large-Scale Data Warehousing Systems, Social Media Data Analytics Platforms

Non-Functional Requirement
Role: System
Requirement ID: NFR-02
Title: Stream Data
Description: The capability to process data which is potentially infinite and usually flowing at high velocity, e.g. monitoring data captured and processed in real-time or close to real-time, using frameworks like Puma, Swift, and Stylus at Facebook, Storm and Heron at Twitter, and Mantis at Netflix.
Citation: Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1, pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840 [Section 3.2]
Related Title: Real-Time Event Processing Systems, IoT Data Streaming Platforms

Non-Functional Requirement
Role: System
Requirement ID: NFR-03
Title: Late and Out of Order Data
Description: The capability to process data which arrives late or in a different order from that in which it was emitted, particularly in stream processing, using mechanisms like low watermarks in Stylus at Facebook, withLag() method in Heron at Twitter, and unspecified strategies in Mantis at Netflix.
Citation: Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1, pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840 [Section 3.3]
Related Title: Event-Driven Streaming Analytics, Mobile Data Ingestion Systems

Non-Functional Requirement
Role: System
Requirement ID: NFR-04
Title: Processing Guarantees
Description: The capability to offer processing guarantees such as exactly once, at least once, and at most once in stream systems, with exactly once semantics provided through checkpointing and idempotency, as implemented in Stylus at Facebook, Heron at Twitter, and Kafka at Netflix.
Citation: Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1, pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840 [Section 3.4]
Related Title: High-Reliability Data Pipeline Systems, Financial Transaction Processing Platforms

Non-Functional Requirement
Role: System
Requirement ID: NFR-05
Title: Integration and Extensibility
Description: The capability to integrate with existing services and components, and to facilitate extension of the architecture to incorporate different components, achieved through messaging systems like Scribe at Facebook, modular design in Heron at Twitter, and rigid but efficient setups in Netflix's architecture.
Citation: Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Related Title: Modular Microservices Architectures, Enterprise Integration Platforms

Non-Functional Requirement
Role: System
Requirement ID: NFR-06
Title: Distribution and Scalability
Description: The capability to distribute data processing amongst different machines in multi-clustered architectures, including dynamic scaling without downtime, implemented via Scribe at Facebook, Dhalion in Heron at Twitter, and Fenzo for elastic resource management at Netflix.
Citation: Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Non-Functional Requirement
Role: System
Requirement ID: NFR-08
Title: Fault Tolerance
Description: The capability of a system to continue operating should one or more nodes fail, recovering gracefully with minimal impact, achieved through node independence and checkpointing at Facebook, modular distributed architecture at Twitter, and multi-regional redundancy at Netflix.
Citation: Vergilio, T. and Ramachandran, M. (2019). Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Non-Functional Requirement
Role: System
Requirement ID: NFR-09
Title: Flow Control
Description: The capability to handle scenarios where the data source emits records faster than the system can consume, using strategies like backpressure management, implemented via Scribe at Facebook, Stream Manager in Heron at Twitter, and ReactiveX libraries in Mantis at Netflix.

Non-Functional Requirement
Role: System
Requirement ID: NFR-10
Title: Flexibility and Technology Agnosticism
Description: The capability of an architecture to use different technologies in place of existing components, with varying degrees of flexibility, least at Facebook due to in-house tools, high at Twitter with Heron's modular design, and emerging at Netflix through containerization and open-source initiatives.
