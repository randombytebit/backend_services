Non-Functional Requirement
Role: System
Requirement ID: NFR-01
Title: Batch Data
Description: The system must support the processing of finite and large-volume data, such as data archived in distributed file systems or databases, with capabilities to handle volumes reaching hundreds of petabytes daily, using combinations of systems like Operational Data Store (ODS), Scuba, Hive, and Laser for storage and processing via tools like Scalding, Presto, and Genie.
Citation: Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840
Related Title: Large-Scale Data Archiving Systems, Distributed File System Processing

Non-Functional Requirement
Role: System
Requirement ID: NFR-02
Title: Stream Data
Description: The system must enable the processing of potentially infinite and high-velocity data in real-time or near real-time, such as monitoring data, utilizing suites of applications like Puma, Swift, and Stylus for stream processing, or frameworks like Storm, Heron, and Mantis integrated with cluster managers like Mesos and Aurora.

Non-Functional Requirement
Role: System
Requirement ID: NFR-03
Title: Late and Out of Order Data
Description: The system must handle stream data that arrives late or out of order, such as mobile user data delayed by reception issues, by employing mechanisms like low watermarks in Stylus, windowing concepts in Heron with methods like withLag(), and potentially similar strategies in Mantis, transforming infinite streams into finite windows for processing.
Citation: Vergilio, T and Ramachandran, M (2019) Non-functional requirements for real world big data systems: an investigation of big data architectures at Facebook, Twitter and Netflix. Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840
Related Title: Mobile Data Stream Handling Systems, Out-of-Order Event Processing Frameworks

Non-Functional Requirement
Role: System
Requirement ID: NFR-04
Title: Processing Guarantees
Description: The system must provide processing guarantees such as exactly once, at least once, or at most once for stream processing, with exactly once achieved through checkpointing and state persistence, at least once via optimizations in Stylus with Swift and Scribe, effectively once semantics in Heron, and exactly once via Kafka's idempotency and atomic transactions in Netflix's setup.

Proceedings of the 13th International Conference on Software Technologies, 1. pp. 833-840. DOI: https://doi.org/10.5220/0006825408330840

Requirement ID: NFR-06

Requirement ID: NFR-08
Title: Fault Tolerance
Description: The system must continue operating with minimal impact on user experience if nodes fail, recovering gracefully through node independence and messaging like Scribe with Swift for checkpointing in Facebook, modular distributed architectures with decoupled scheduling and JVM isolation in Heron, and cloud-based redundancy via Active-Active deployments across regions in Netflix.

Non-Functional Requirement
Role: System
Requirement ID: NFR-09
Title: Flow Control
Description: The system must manage scenarios where data sources emit records faster than consumption, using backpressure strategies like persistent messaging in Scribe for node reallocation in Facebook, tightening upstream components in Heron to match slowest processing, and programmatic methods in Mantis via ReactiveX libraries like buffering, sampling, or windowing in RxJava.
