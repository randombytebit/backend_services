Non-Functional Requirement  
Role: System  
Requirement ID: NFR-01  
Title: Batch Data  
Description: System should have the capability to process data which is finite and usually large in volume, e.g. data archived in distributed file systems or databases, with examples including hundreds of petabytes stored and tens of petabytes processed daily.  
Citation: Vergilio and Ramachandran (2019), Section 3.1  
Related Title: Social Media Batch Analytics Pipelines, Distributed File System Processing  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-02  
Title: Stream Data  
Description: System should have the capability to process data which is potentially infinite and usually flowing at high velocity, e.g. monitoring data captured and processed in real-time or close to real-time.  
Citation: Vergilio and Ramachandran (2019), Section 3.2  
Related Title: Real-Time User Activity Streams, High-Velocity Monitoring Systems  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-03  
Title: Late and Out of Order Data  
Description: System should have the capability to process data which arrives late or in a different order from that in which it was emitted, such as streaming data from mobile users delayed by reception loss, often using concepts like windows of data, low watermarks, or lag methods.  
Citation: Vergilio and Ramachandran (2019), Section 3.3  
Related Title: Mobile Streaming with Delays, Out-of-Order Event Processing  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-04  
Title: Processing Guarantees  
Description: System should offer processing guarantees such as exactly once, at least once, or at most once semantics for stream processing, balancing completeness, duplication avoidance, and latency costs through mechanisms like checkpointing, idempotency, or effectively once delivery.  
Citation: Vergilio and Ramachandran (2019), Section 3.4  
Related Title: Reliable Stream Semantics in Analytics, Duplicate-Free Data Pipelines  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-05  
Title: Integration and Extensibility  
Description: System should integrate with existing services and components, and provide provisions to facilitate extension of the architecture to incorporate different components in the future, such as through shared messaging systems or modular designs.  
Citation: Vergilio and Ramachandran (2019), Section 3.5  
Related Title: Modular Streaming Frameworks, Service Pipeline Integration  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-06  
Title: Distribution and Scalability  
Description: System should distribute data processing amongst different machines in multi-clustered architectures across data centres, and support dynamic scaling by adding or removing nodes without downtime, using tools for parallelism and resource adjustment.  
Citation: Vergilio and Ramachandran (2019), Section 3.6  
Related Title: Multi-Cluster Data Distribution, Dynamic Node Scaling in Big Data  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-07  
Title: Cloud Support and Elasticity  
Description: System should support deployment into the cloud to leverage economies of scale and elasticity, allowing scaling up and down according to demand with pay-per-use resources, potentially enhanced by containerization to reduce vendor lock-in.  
Citation: Vergilio and Ramachandran (2019), Section 3.7  
Related Title: Elastic Cloud-Based Analytics, Multi-Cloud Container Deployments  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-08  
Title: Fault Tolerance  
Description: System should continue to operate if one or more nodes fail, recovering gracefully with minimal impact on user experience, through mechanisms like node independence, checkpointing, modular designs, decentralized management, or multi-region redundancy.  
Citation: Vergilio and Ramachandran (2019), Section 3.8  
Related Title: Resilient Multi-Region Deployments, Node Failure Recovery in Clusters  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-09  
Title: Flow Control  
Description: System should handle scenarios where data sources emit records faster than consumption, using backpressure strategies such as node reallocation, upstream throttling, buffering, sampling, or reactive library constructs to prevent overload.  
Citation: Vergilio and Ramachandran (2019), Section 3.9  
Related Title: Backpressure in High-Throughput Streams, Reactive Data Flow Management  

Non-Functional Requirement  
Role: System  
Requirement ID: NFR-10  
Title: Flexibility and Technology Agnosticism  
Description: System should allow use of different technologies in place of existing components, achieved through modular architectures, open-source components, container abstractions, or standardized interfaces to avoid tight coupling and enable replacements.  
Citation: Vergilio and Ramachandran (2019), Section 3.10  
Related Title: Modular Open-Source Frameworks, Containerized Technology Swaps
